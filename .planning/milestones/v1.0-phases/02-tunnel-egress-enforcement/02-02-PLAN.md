实现以下函数：

**`GetContainerNetNS(containerName string) (netns.NsHandle, uint32, error)`**
- 调用 `docker inspect -f '{{.State.Pid}}' <containerName>` 获取 PID
- 验证 PID > 0（否则容器未运行）
- 调用 `netns.GetFromPid(pid)` 获取 netns fd
- 返回 (nsHandle, pid, nil)
- 出错时返回包装的 `NetworkError{Type: ErrTunnelSetupFailed}`

**`InjectManagementVeth(hostNS, containerNS netns.NsHandle, hostID string) (hostIP, containerIP string, err error)`**
- `runtime.LockOSThread()` + `defer runtime.UnlockOSThread()`
- 从 hostID 前 4 字节派生管理网段索引（避免冲突）：`idx := binary.BigEndian.Uint16([]byte(hostID[:4])) % 16382`；管理 IP 对为 `10.99.{idx/128 + 1}.{(idx%128)*2 + 1}/30` 和 `.{(idx%128)*2 + 2}/30`
- 如果上述方案过于复杂，可简化为使用 hostID 的哈希值取模分配唯一 /30 网段
- veth 名称：`mgmt-{hostID[:8]}` (宿主机侧) 和 `mgmt0` (容器侧)
- 在宿主机 netns 中创建 veth pair：`netlink.LinkAdd(&netlink.Veth{PeerName: peerName, ...})`
- 将 peer 端移入容器 netns：`netlink.LinkSetNsFd(peer, int(containerNS))`
- 在宿主机侧为 mgmt 接口配 IP 并 up
- 切到容器 netns，为 mgmt0 配 IP 并 up（不配默认网关！——防止 veth 成为旁路出网路径）
- 切回宿主机 netns
- 返回 (hostIP, containerIP, nil)

package network

import (
    "fmt"
    "net"
    "runtime"

    "github.com/vishvananda/netlink"
    "github.com/vishvananda/netns"
    "golang.zx2c4.com/wireguard/wgctrl"

)

实现以下函数：

**`GenerateWireGuardKeys() (privateKey, publicKey string, err error)`**
- 调用 `wgtypes.GeneratePrivateKey()`
- 返回 `(privKey.String(), privKey.PublicKey().String(), nil)`

**`InjectWireGuard(hostNS, containerNS netns.NsHandle, spec TunnelSpec) error`**
- `runtime.LockOSThread()` + `defer runtime.UnlockOSThread()`
- 确保当前在宿主机 netns

**步骤 a：在宿主机 netns 创建 WireGuard 接口**
- 接口名使用 `spec.InterfaceName`（如 `wg-<hostID[:8]>`）
- `netlink.LinkAdd(&netlink.Wireguard{LinkAttrs: netlink.LinkAttrs{Name: spec.InterfaceName}})`

**步骤 b：配置 WireGuard**
- 解析 `spec.PrivateKey` → `wgtypes.ParseKey(spec.PrivateKey)`
- 解析 `spec.PeerPublicKey` → `wgtypes.ParseKey(spec.PeerPublicKey)`
- 如果 `spec.PresharedKey` 非空，解析之
- 解析 `spec.AllowedIPs` → `net.ParseCIDR` 得到 `[]net.IPNet`
- 构建 `wgtypes.Config{PrivateKey, Peers: [{PublicKey, Endpoint: spec.PeerEndpoint, AllowedIPs, PersistentKeepaliveInterval: 25s}]}`
- `client.ConfigureDevice(spec.InterfaceName, config)`

**步骤 c：移入容器 netns**
- `link, _ := netlink.LinkByName(spec.InterfaceName)`
- `netlink.LinkSetNsFd(link, int(containerNS))`
- WireGuard 的加密 UDP socket 自动留在宿主机侧（WireGuard 官方 birthplace namespace 机制）

**步骤 d：在容器 netns 内配地址和路由**
- `netns.Set(containerNS)` + `defer netns.Set(hostNS)`
- 获取容器内的 wg 接口（移入后名称可能需要确认）
- `netlink.AddrAdd(link, &netlink.Addr{IPNet: spec.TunnelAddress})`
- `netlink.LinkSetUp(link)`
- 添加默认路由指向 wg 接口：`netlink.RouteAdd(&netlink.Route{LinkIndex: link.Attrs().Index, Dst: &net.IPNet{IP: net.IPv4zero, Mask: net.CIDRMask(0, 32)}})`

每个步骤出错时返回 `&NetworkError{Type: ErrTunnelSetupFailed, Message: "...: <具体错误>"}`。

package network

import (
    "fmt"
    "runtime"

    "github.com/google/nftables"
    "github.com/google/nftables/expr"
    "github.com/vishvananda/netns"
)

实现：

**`ApplyFirewallRules(containerNS netns.NsHandle, wgIfIndex, loIfIndex, mgmtIfIndex int) error`**

- `runtime.LockOSThread()` + `defer runtime.UnlockOSThread()`
- 创建 nftables 连接：`nftables.New(nftables.WithNetNSFd(int(containerNS)))`

**IPv4 规则（family: TableFamilyIPv4）：**
- 创建 table `filter`
- 创建 chain `output`：type filter, hook output, priority 0, policy **DROP**（默认拒绝）
- Rule 1：允许 loopback 出站 — `oif lo → accept`
- Rule 2：允许 WireGuard 接口出站 — `oif wgIfIndex → accept`
- Rule 3：允许管理 veth 上的 established/related 回包 — `oif mgmtIfIndex + ct state established,related → accept`
- 创建 chain `input`：type filter, hook input, priority 0, policy **DROP**
- Rule 4：允许 loopback 入站 — `iif lo → accept`
- Rule 5：允许 WireGuard 接口入站（established + new，支持隧道流量）— `iif wgIfIndex → accept`
- Rule 6：允许管理 veth SSH 入站（目标端口 22）— `iif mgmtIfIndex + tcp dport 22 → accept`
- Rule 7：允许管理 veth established/related 入站 — `iif mgmtIfIndex + ct state established,related → accept`

**IPv6 规则（family: TableFamilyIPv6）：**
- 创建 table `filter6`
- 创建 chain `output6`：policy **DROP**（全部拒绝，仅允许 ip6 loopback）
- Rule 1：允许 loopback — `oif lo → accept`
- 所有其他 IPv6 出站被拒绝，防止 IPv6 绕过隧道

- 调用 `conn.Flush()` 提交所有规则
- 出错时返回 `&NetworkError{Type: ErrTunnelSetupFailed, Message: "apply firewall: ..."}`

**注意：** nftables `expr` 包构建规则需要使用 `expr.Meta`（oif/iif）、`expr.Cmp`（比较）、`expr.Ct`（conntrack state）、`expr.Verdict`（accept）等表达式原语。参考 `google/nftables` 仓库的 nftables_test.go 了解具体用法。

**`TunnelProvider.PrepareHost(ctx context.Context, spec HostNetworkSpec) error`：**

完整管线（按严格顺序执行，任一步失败立即返回错误，不做自动重试 per D-04）：

```
1. 前置检查
   - spec.Egress 不能为 nil（调用方应已完成绑定校验）
   - 如果 spec.Egress.Tunnel.PrivateKey 为空，调用 GenerateWireGuardKeys() 生成
     （密钥持久化由调用方在 worker 层处理，Provider 只使用密钥）

2. 获取容器 netns
   - containerName := fmt.Sprintf("cloudproxy-%s", spec.HostID)
   - nsHandle, pid, err := GetContainerNetNS(containerName)
   - defer nsHandle.Close()
   - spec.ContainerPID = pid

3. 获取宿主机 netns
   - hostNS, err := netns.Get()
   - defer hostNS.Close()

4. 注入管理 veth 对
   - hostIP, containerIP, err := InjectManagementVeth(hostNS, nsHandle, spec.HostID)
   - 记录日志：logger.Info("management veth injected", "host_ip", hostIP, "container_ip", containerIP)

5. 注入 WireGuard 隧道
   - 设置 InterfaceName：spec.Egress.Tunnel.InterfaceName = fmt.Sprintf("wg-%s", spec.HostID[:8])
   - err := InjectWireGuard(hostNS, nsHandle, spec.Egress.Tunnel)

6. 配置 DNS
   - err := ConfigureContainerDNS(pid, spec.Egress.Tunnel.DNSServer)

7. 应用防火墙规则
   - 获取容器内接口 index（需在容器 netns 内查询）
   - err := ApplyFirewallRules(nsHandle, wgIfIndex, loIfIndex, mgmtIfIndex)

8. 返回 nil（全部成功）
```

**`TunnelProvider.CleanupHost(ctx context.Context, spec HostNetworkSpec) error`：**

- 尝试删除宿主机侧的管理 veth 接口 `mgmt-{hostID[:8]}`（如果存在）
- 尝试删除宿主机侧的 WireGuard 接口 `wg-{hostID[:8]}`（如果存在，通常已被移入容器 netns）
- 容器停止后其 netns 自动销毁，netns 内的接口和 nftables 规则也随之清除
- 清理操作出错只记录日志，不返回错误（防止清理失败阻塞 rebuild）
