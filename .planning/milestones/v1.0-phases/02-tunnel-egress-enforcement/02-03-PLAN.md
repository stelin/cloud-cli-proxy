定义校验结果结构体：

```go
type VerifyResult struct {
    EgressIPMatch  bool   // 出口 IP 与绑定结果是否一致
    ActualEgressIP string // 实际检测到的出口 IP
    DNSCorrect     bool   // DNS 是否走受控路径
    ActualDNS      string // 实际使用的 DNS 服务器
    LeakBlocked    bool   // 非隧道直连是否被阻断
    LeakTarget     string // 泄漏测试使用的目标地址
}

func (r VerifyResult) AllPassed() bool {
    return r.EgressIPMatch && r.DNSCorrect && r.LeakBlocked
}
```

实现主校验函数：

**`VerifyNetworkIntegrity(ctx context.Context, containerPID uint32, expected EgressConfig) (VerifyResult, error)`**

使用 `nsenter -t <pid> -n --` 前缀在容器 netns 内执行命令：

```go
nsenterPrefix := []string{"nsenter", "-t", strconv.FormatUint(uint64(containerPID), 10), "-n", "--"}
```

**校验 1：出口 IP 匹配（D-09 第一项）**
- 执行命令：`nsenter -t <pid> -n -- curl -4 --max-time 10 -s https://api.ipify.org`
- 超时上下文：10 秒
- 解析输出：`strings.TrimSpace(output)` 得到实际出口 IP
- 比较：`actualIP == expected.ExpectedIP`
- 如果 curl 执行失败（超时或网络不可达）：
  - `result.EgressIPMatch = false`
  - 返回 `&NetworkError{Type: ErrEgressUnreachable, HostID: 从 ctx 获取, Metadata: map[string]any{"endpoint": expected.Tunnel.PeerEndpoint.String()}}`
- 如果 IP 不匹配：
  - `result.EgressIPMatch = false`
  - 不立即返回错误，继续后续校验以收集完整状态

**校验 2：DNS 路径正确（D-09 第二项）**
- 执行命令：`nsenter -t <pid> -n -- cat /etc/resolv.conf`
- 解析输出：提取 `nameserver` 行的 IP 地址（取第一个）
- 比较：`actualDNS == expected.Tunnel.DNSServer`
- 如果不一致：`result.DNSCorrect = false`
- 额外验证（可选但推荐）：执行 `nsenter -t <pid> -n -- nslookup -timeout=5 example.com <expected.Tunnel.DNSServer>`
  - 如果失败，说明 DNS 服务器不可达或无响应

**校验 3：泄漏阻断（D-09 第三项）**
- 目标：`1.1.1.1:80`（Cloudflare DNS，用作非隧道直连测试目标）
- 执行命令：`nsenter -t <pid> -n -- timeout 3 bash -c "echo | nc -w 2 1.1.1.1 80"`
  - 此命令尝试绕过 WireGuard 直接连接外部 IP
- 预期行为：连接失败（被 nftables 阻断）
- `result.LeakBlocked = (err != nil)` — 命令失败表示泄漏被成功阻断
- `result.LeakTarget = "1.1.1.1:80"`
- 注意：此测试依赖容器内安装了 `nc`（netcat）。如果受管镜像未包含 nc，改用 Go 标准库在容器 netns 内尝试 TCP 连接：
  ```go
  // 备选方案：使用 nsenter + bash TCP 重定向
  "nsenter -t <pid> -n -- timeout 3 bash -c 'echo >/dev/tcp/1.1.1.1/80'"
  ```

**汇总判定：**
- 如果 `!result.AllPassed()`，返回对应类型的 `NetworkError`：
  - `!EgressIPMatch && ActualEgressIP != ""` → `ErrEgressIPMismatch` with metadata `{"expected": expected.ExpectedIP, "actual": result.ActualEgressIP}`
  - `!EgressIPMatch && ActualEgressIP == ""` → `ErrEgressUnreachable`
  - `!DNSCorrect` → `ErrDNSLeak` with metadata `{"expected_dns": expected.Tunnel.DNSServer, "actual_dns": result.ActualDNS}`
  - `!LeakBlocked` → `ErrLeakNotBlocked` with metadata `{"target": result.LeakTarget}`
  - 如果多个校验同时失败，返回第一个（按优先级：EgressIP > DNS > Leak）
