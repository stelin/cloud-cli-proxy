    2) 在 `queries.go` 增加：
       - `GetBootstrapUserByUsername(ctx, username)`：读取 `users.username/password_hash/status`。
       - `GetPrimaryHostByUserID(ctx, userID)`：读取该用户可启动主机（优先 `slot_key='primary'`，无记录时返回 not found）。
    3) 新建 `bootstrap_auth.go`，实现 `POST /v1/bootstrap/sessions`：
       - 请求体：`{username,password}`。
       - 用 `bcrypt.CompareHashAndPassword` 校验密码（per D-03）。
       - 账号状态仅允许 `active`；`disabled/expired` 直接失败（per D-03, D-10）。
       - 认证通过后调用 `QueueHostAction(..., ActionStartHost, "bootstrap:"+username)` 入队（per D-04）。
       - 返回结构包含 `task_id`、`status_url`、`stage_code=auth_passed`、`stage_text=认证通过，主机启动中`。
       - 失败统一返回稳定 `error_code` 与中文 `message`，且不自动重试（per D-10, D-11）。
    4) 新建 `bootstrap_auth_test.go`（表驱动）覆盖：
       - 凭证错误 -> `auth_invalid`，不调用 QueueHostAction。
       - 账号禁用/过期 -> `account_disabled/account_expired`，不调用 QueueHostAction。
       - 认证成功 -> 调用一次 `QueueHostAction(ActionStartHost)` 并返回 task_id。
  </action>
