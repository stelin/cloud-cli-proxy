**总体模式（per D-08, D-09）：** 每个需要事件记录的 handler 新增一个 `EventRecorder` 接口依赖。统一定义在 router.go 中：

```go
type EventRecorder interface {
    RecordEvent(context.Context, repository.RecordEventParams) (repository.Event, error)
}
```

事件记录在主操作**成功后**执行（fire-and-forget 风格）。事件记录失败只记日志，不影响主操作返回值。

```go
func (c *Client) InspectContainer(ctx context.Context, containerName string) (ContainerStatusResponse, error) {
    req, err := http.NewRequestWithContext(ctx, http.MethodGet,
        fmt.Sprintf("http://host-agent/v1/containers/%s/status", containerName), nil)
    if err != nil {
        return ContainerStatusResponse{}, fmt.Errorf("build inspect request: %w", err)
    }

    response, err := c.httpClient.Do(req)
    if err != nil {
        return ContainerStatusResponse{}, fmt.Errorf("inspect container via %s: %w", c.socketPath, err)
    }
    defer response.Body.Close()

    var payload ContainerStatusResponse
    if err := json.NewDecoder(response.Body).Decode(&payload); err != nil {
        return ContainerStatusResponse{}, fmt.Errorf("decode inspect response: %w", err)
    }

    return payload, nil
}
```

```go
mux.HandleFunc("GET /v1/containers/{name}/status", func(w http.ResponseWriter, r *http.Request) {
    containerName := r.PathValue("name")

    // 复用 worker 中 containerExists 的逻辑（docker container inspect）
    cmd := exec.CommandContext(r.Context(), "docker", "container", "inspect",
        "-f", "{{.State.Running}}", containerName)
    out, err := cmd.Output()
    if err != nil {
        // 容器不存在或命令失败
        writeJSON(w, http.StatusOK, agentapi.ContainerStatusResponse{
            Name:    containerName,
            Exists:  false,
            Running: false,
        })
        return
    }

    running := strings.TrimSpace(string(out)) == "true"
    writeJSON(w, http.StatusOK, agentapi.ContainerStatusResponse{
        Name:    containerName,
        Exists:  true,
        Running: running,
    })
})
```

**4. 创建 `internal/controlplane/scheduler/reconciler.go`（per D-15, D-16, D-17, D-18）：**

```go
package scheduler

import (
    "context"
    "fmt"
    "log/slog"
    "time"
    "github.com/zaneliu/cloud-cli-proxy/internal/agentapi"
    "github.com/zaneliu/cloud-cli-proxy/internal/store/repository"
)

type ReconcileStore interface {
    ListRunningHosts(context.Context) ([]repository.Host, error)
    UpdateHostStatus(context.Context, string, string) error
    MarkStaleTasks(context.Context, time.Duration) ([]repository.Task, error)
    RecordEvent(context.Context, repository.RecordEventParams) (repository.Event, error)
}

type ContainerInspector interface {
    InspectContainer(context.Context, string) (agentapi.ContainerStatusResponse, error)
}

type Reconciler struct {
    logger         *slog.Logger
    store          ReconcileStore
    inspector      ContainerInspector
    staleThreshold time.Duration
}

func NewReconciler(logger *slog.Logger, store ReconcileStore, inspector ContainerInspector, staleThreshold time.Duration) *Reconciler {
    if staleThreshold == 0 {
        staleThreshold = 10 * time.Minute
    }
    return &Reconciler{
        logger: logger, store: store, inspector: inspector, staleThreshold: staleThreshold,
    }
}
```

`Run` 方法逻辑：

**Step A: 主机漂移检测（per D-15）**
1. 调用 `store.ListRunningHosts(ctx)` 获取 DB 中 status='running' 的主机
2. 对每个主机，构造容器名 `cloudproxy-{host.ID}`（与 runtime_service.go 中 `containerNameForHost` 一致），调用 `inspector.InspectContainer(ctx, containerName)`
3. 严格区分通信失败和容器不存在（per RESEARCH Pitfall 4）：
   - 如果 `InspectContainer` 返回 error → 这是通信失败，只记日志，**不修改 DB 状态**
   - 如果返回成功但 `Exists=false` 或 `Running=false` → 容器不存在或已停止，调用 `store.UpdateHostStatus(ctx, host.ID, "stopped")`，然后记录 `reconcile.host.drift` 事件：
     ```go
     store.RecordEvent(ctx, repository.RecordEventParams{
         HostID:  &host.ID,
         Level:   "warn",
         Type:    "reconcile.host.drift",
         Message: "对账发现主机状态漂移",
         Metadata: map[string]any{
             "operator":      "system",
             "db_status":     "running",
             "actual_status": actualStatus, // "stopped" 或 "not_found"
             "host_id":       host.ID,
         },
     })
     ```
4. 单个主机处理失败不中断整个扫描

**Step B: 陈旧任务清理（per D-16）**
1. 调用 `store.MarkStaleTasks(ctx, r.staleThreshold)`，获取被标记为 failed 的陈旧任务列表
2. 对每个陈旧任务记录 `reconcile.task.stale` 事件：
   ```go
   store.RecordEvent(ctx, repository.RecordEventParams{
       HostID:  task.HostID,
       Level:   "warn",
       Type:    "reconcile.task.stale",
       Message: "对账发现陈旧任务",
       Metadata: map[string]any{
           "operator":    "system",
           "task_id":     task.ID,
           "kind":        task.Kind,
           "old_status":  "pending/running",
           "threshold":   r.staleThreshold.String(),
       },
   })
   ```
