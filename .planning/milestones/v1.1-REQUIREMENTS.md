# Requirements: Cloud CLI Proxy

**Defined:** 2026-03-28
**Core Value:** 给每个用户提供一台开箱即用的 SSH 云主机，并且严格保证其所有出网流量都走受控的指定出口 IP，同时保持"一条命令启动"的体验足够顺滑。

## v1.1 Requirements

Requirements for v1.1 milestone: 支持代理协议出网。Each maps to roadmap phases.

### 数据层与类型化

- [x] **DATA-01**: 管理员可以创建出口 IP 时选择隧道类型（wireguard 或 proxy）
- [x] **DATA-02**: 代理类型的出口 IP 可以存储 sing-box outbound JSON 配置（proxy_config）
- [x] **DATA-03**: 绑定校验根据 tunnel_type 分支验证配置完整性（WG 要求 wg_* 字段，proxy 要求 proxy_config 非空且 JSON 合法）
- [x] **DATA-04**: 代理类型的出口 IP 配置校验白名单 outbound type（socks/vmess/shadowsocks/trojan/http）并验证必需字段
- [x] **DATA-05**: DB migration 确保现有 WireGuard 出口 IP 自动获得 tunnel_type='wireguard' 默认值，不影响已有数据

### SingBox Provider

- [x] **SING-01**: 代理类型的主机启动时，SingBoxProvider 在容器 netns 内启动 sing-box tun 模式进程，实现全流量代理
- [x] **SING-02**: sing-box 配置确保 DNS 查询走代理路径，不使用系统 DNS，不发生 DNS 泄漏
- [x] **SING-03**: nftables 规则允许 tun0 接口流量（替代 wg0），保持默认拒绝策略
- [x] **SING-04**: 三重校验（出口 IP 匹配 + DNS 路径 + 泄漏阻断）对代理类型同样生效
- [x] **SING-05**: 受管用户镜像预装 sing-box 二进制
- [x] **SING-06**: Provider 工厂根据 egressConfig.TunnelType 自动选择 TunnelProvider 或 SingBoxProvider
- [x] **SING-07**: 容器停止或重建时 sing-box 进程被正确清理，无僵尸进程

### 代理测试

- [x] **TEST-01**: 管理员可以对任意出口 IP 执行一键测试，验证连通性（HTTP 204 检测）
- [x] **TEST-02**: 测试包含出口 IP 匹配检测（通过代理请求 ipify/ip.me/ifconfig.me，对比预期 IP）
- [x] **TEST-03**: 测试包含 DNS 泄漏检测（验证 DNS 查询走代理路径，非宿主机本地 DNS）
- [x] **TEST-04**: 测试 API 设置合理超时（总超时 30s），代理不可达时在有限时间内返回错误

### 前端适配

- [x] **UI-01**: 出口 IP 创建/编辑表单根据 tunnel_type 切换显示 WireGuard 字段或代理协议字段
- [x] **UI-02**: 代理协议模式下支持按协议类型（socks/vmess/ss/trojan/http）动态渲染对应字段
- [x] **UI-03**: 提供高级 JSON 编辑模式，直接编辑 sing-box outbound 配置
- [x] **UI-04**: 出口 IP 列表页/详情页提供测试按钮，显示 loading → 结果（pass/fail + 各项详情）
- [x] **UI-05**: 出口 IP 列表页用状态指示器显示最近一次测试结果

## Future Requirements

### 代理增强

- **PROX-01**: 代理健康定时检测 + 告警通知
- **PROX-02**: 代理配置模板（常用提供商预设）
- **PROX-03**: 批量代理测试
- **PROX-04**: 多节点负载均衡（sing-box urltest/fallback）
- **PROX-05**: 代理流量统计和监控

## Out of Scope

| Feature | Reason |
|---------|--------|
| 用户自选代理节点 | 由管理员统一配置，避免安全和支持风险 |
| 代理链/多跳 | 延迟增加、排查困难，单跳足够 |
| 实时流量监控 | 开发量大，v1.1 先做连通性测试 |
| 自动代理节点切换 | 复杂度高，先单节点稳定 |
