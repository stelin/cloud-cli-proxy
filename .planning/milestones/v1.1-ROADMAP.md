## Phase Details

### Phase 7: 数据层与类型化
**Goal**: 出口 IP 数据模型完整支持 wireguard 和 proxy 两种隧道类型，校验和 API 按类型分支工作
**Depends on**: Phase 6 (v1.0 MVP)
**Requirements**: DATA-01, DATA-02, DATA-03, DATA-04, DATA-05
**Success Criteria** (what must be TRUE):
  1. 管理员创建出口 IP 时可以选择 wireguard 或 proxy 隧道类型
  2. proxy 类型的出口 IP 能够存储完整的 sing-box outbound JSON 配置
  3. 绑定校验根据隧道类型分支验证——WG 要求 wg_* 字段完整，proxy 要求 proxy_config 非空且 JSON 合法
  4. proxy_config 白名单限制 outbound type（socks/vmess/shadowsocks/trojan/http）并验证协议必需字段，拒绝非法配置
  5. 已有的 WireGuard 出口 IP 在 DB migration 后自动获得 tunnel_type='wireguard' 默认值，无数据破坏
**Plans**: 3 plans

Plans:
- [x] 07-01-PLAN.md — DB migration + Repository 模型和 SQL 查询扩展
- [x] 07-02-PLAN.md — Network 类型扩展 + ValidateEgressBinding 分支
- [x] 07-03-PLAN.md — API handler 适配 + proxy_config 校验 + Worker 适配

### Phase 8: SingBoxProvider 与受管镜像
**Goal**: 代理类型的主机能通过 sing-box tun 模式实现全流量代理出网，安全性和校验标准等同 WireGuard
**Depends on**: Phase 7
**Requirements**: SING-01, SING-02, SING-03, SING-04, SING-05, SING-06, SING-07
**Success Criteria** (what must be TRUE):
  1. 代理类型主机启动后，所有出站流量（含 DNS）通过 sing-box tun 接口路由，无泄漏
  2. Provider 工厂根据 tunnel_type 自动选择 WireGuard 或 sing-box，现有 WireGuard 路径不受影响
  3. 受管镜像内预装 sing-box 二进制，无需运行时下载
  4. 容器停止或重建时 sing-box 进程被正确清理，无僵尸进程
  5. 三重校验（出口 IP 匹配 + DNS 路径 + 泄漏阻断）对代理类型同样生效
**Plans**: 3 plans

Plans:
- [x] 08-01-PLAN.md — sing-box 配置模型 + 受管镜像预装
- [x] 08-02-PLAN.md — Proxy 防火墙规则 + 宿主机 IP 转发
- [x] 08-03-PLAN.md — SingBoxProvider 核心实现 + RoutingProvider 工厂 + 注入

### Phase 9: 前端适配与代理测试
**Goal**: 管理后台完整支持代理类型出口 IP 的配置、验证和测试结果展示
**Depends on**: Phase 8
**Requirements**: TEST-01, TEST-02, TEST-03, TEST-04, UI-01, UI-02, UI-03, UI-04, UI-05
**Success Criteria** (what must be TRUE):
  1. 出口 IP 创建/编辑表单根据隧道类型动态切换 WireGuard 字段或代理协议字段
  2. 代理协议模式下按协议类型（socks/vmess/ss/trojan/http）动态渲染对应字段，并支持高级 JSON 编辑模式
  3. 管理员可以对任意出口 IP 执行一键测试，30 秒超时内返回连通性、出口 IP 匹配和 DNS 泄漏检测结果
  4. 测试结果在出口 IP 列表页以状态指示器展示，详情页显示各项检测明细
**Plans**: 3 plans
**UI hint**: yes

Plans:
- [x] 09-01-PLAN.md — 后端代理测试 API + 前端 TypeScript 类型和 hook
- [x] 09-02-PLAN.md — 动态表单重构（隧道类型切换 + 代理协议字段 + JSON 模式）
- [x] 09-03-PLAN.md — 列表页增强（隧道类型列 + 测试状态列 + 测试结果 Dialog）
