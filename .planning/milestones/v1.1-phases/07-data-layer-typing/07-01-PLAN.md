ALTER TABLE egress_ips
    ADD COLUMN tunnel_type TEXT NOT NULL DEFAULT 'wireguard';

ALTER TABLE egress_ips
    ADD CONSTRAINT egress_ips_tunnel_type_check
    CHECK (tunnel_type IN ('wireguard', 'proxy'));

ALTER TABLE egress_ips
    ADD COLUMN proxy_config JSONB;

- CHECK 约束限制 `tunnel_type` 只能是 `'wireguard'` 或 `'proxy'`（per D-02, D-23 第一层）。

**queries.go 变更（需要更新 6 个方法的 SQL）：**

以下每个方法的 SELECT 列表中，在 `status` 之后追加 `tunnel_type`，在 `wg_peer_address::text` 之后追加 `proxy_config`。对应的 Scan 调用中，在 `&item.Status` 之后追加 `&item.TunnelType`，在 `&item.WgPeerAddress` 之后追加 `&item.ProxyConfig`。

1. **`ListEgressIPs`**：SELECT 加 `tunnel_type, ... proxy_config`；Scan 加 `&item.TunnelType, ... &item.ProxyConfig`。

2. **`GetEgressIP`**：同上模式。

3. **`GetEgressIPByHost`**：SELECT 中列使用 `e.` 前缀，加 `e.tunnel_type, ... e.proxy_config`；Scan 同上。

4. **`GetHostDetail`**（内部 JOIN 查询部分）：SELECT 中列使用 `e.` 前缀，加 `e.tunnel_type, ... e.proxy_config`；Scan 中使用 `&b.EgressIP.TunnelType, ... &b.EgressIP.ProxyConfig`。

5. **`CreateEgressIP`**：
   - INSERT 列列表追加 `tunnel_type, proxy_config`
   - VALUES 中追加两个参数占位符 `$10, $11`（原来到 $9）
   - RETURNING 列列表追加 `tunnel_type, proxy_config`
   - 函数参数传递追加 `params.TunnelType, params.ProxyConfig`
   - Scan 追加 `&item.TunnelType, &item.ProxyConfig`
   - **注意**：`tunnel_type` 参数使用 `defaultIfEmpty(params.TunnelType, "wireguard")` 以确保空值时默认为 `"wireguard"`

6. **`UpdateEgressIP`**：
   - UPDATE SET 追加 `tunnel_type = $12, proxy_config = $13`（原来到 $11）
   - RETURNING 列列表追加 `tunnel_type, proxy_config`
   - 函数参数传递追加 `defaultIfEmpty(params.TunnelType, "wireguard"), params.ProxyConfig`
   - Scan 追加 `&item.TunnelType, &item.ProxyConfig`

**关键注意事项：**
- `proxy_config` 在 pgx 中直接 Scan 到 `json.RawMessage`（即 `[]byte`），NULL 值自动映射为 nil，不需要 COALESCE（per Research Pitfall #1）。
- 不要在 SQL 中使用 `COALESCE(proxy_config, '{}'::jsonb)`，否则 WireGuard 行的 proxy_config 会返回空 JSON 对象而非 nil。
- `tunnel_type` 是 NOT NULL 列，直接 Scan 到 `string`。
