From internal/network/validate.go (how ProxySpec is populated):
```go
func validateProxyBinding(record EgressIPRecord, hostID string) (EgressConfig, error) {
	// ...
	var parsed map[string]any
	json.Unmarshal(record.ProxyConfig, &parsed)
	dnsServer, _ := parsed["dns_server"].(string)
	return EgressConfig{
		Proxy: &ProxySpec{
			OutboundConfig: record.ProxyConfig, // includes dns_server field
			DNSServer:      dnsServer,
		},
	}, nil
}
```

IMPORTANT: ProxySpec.OutboundConfig is the ENTIRE proxy_config JSON from DB,
which includes a "dns_server" field alongside sing-box outbound fields.
When building the sing-box outbound, "dns_server" must be stripped (not a valid sing-box field).

**1. sing-box 配置结构体：**

```go
type singBoxConfig struct {
    Log       singBoxLog        `json:"log"`
    DNS       singBoxDNS        `json:"dns"`
    Inbounds  []json.RawMessage `json:"inbounds"`
    Outbounds []json.RawMessage `json:"outbounds"`
    Route     singBoxRoute      `json:"route"`
}

type singBoxLog struct {
    Level string `json:"level"`
}

type singBoxDNS struct {
    Servers  []singBoxDNSServer `json:"servers"`
    Strategy string             `json:"strategy,omitempty"`
}

type singBoxDNSServer struct {
    Tag    string `json:"tag"`
    Type   string `json:"type"`
    Server string `json:"server"`
    Detour string `json:"detour,omitempty"`
}

type singBoxRoute struct {
    Rules            []singBoxRouteRule `json:"rules"`
    DefaultInterface string             `json:"default_interface,omitempty"`
}

type singBoxRouteRule struct {
    Action   string `json:"action,omitempty"`
    Protocol string `json:"protocol,omitempty"`
}
```

**2. `buildSingBoxConfig(spec ProxySpec) ([]byte, error)` 函数：**

构建完整 sing-box 配置 JSON，包含：
- `log.level`: `"warn"`
- `dns.servers`: 一个 server，tag=`"proxy-dns"`, type=`"udp"`, server=`spec.DNSServer`, detour=`"proxy-out"`
- `dns.strategy`: `"ipv4_only"`
- `inbounds`: 一个 tun inbound JSON（通过 marshal map 生成），字段：
  - `type`: `"tun"`, `tag`: `"tun-in"`, `interface_name`: `"tun0"`
  - `address`: `["172.18.0.1/30"]`（注意是 `address` 不是已废弃的 `inet4_address`）
  - `mtu`: `1500`, `auto_route`: `true`, `strict_route`: `true`, `stack`: `"system"`
- `outbounds`: 调用 `buildOutbound(spec.OutboundConfig)` 得到 proxy outbound + 一个 direct outbound（type=direct, tag=direct, bind_interface=mgmt0）
- `route.rules`: `[{"action":"sniff"}, {"protocol":"dns","action":"hijack-dns"}]`
- `route.default_interface`: `"mgmt0"`

最终调用 `json.MarshalIndent(config, "", "  ")` 返回。

**3. `buildOutbound(userConfig json.RawMessage) (json.RawMessage, error)` 函数：**

- 将 `userConfig` unmarshal 为 `map[string]any`
- **删除 `"dns_server"` 键**（这是我们自定义的字段，不是 sing-box outbound 的合法字段）
- 注入 `"tag": "proxy-out"` 和 `"bind_interface": "mgmt0"`
- 重新 marshal 为 JSON 返回

**4. `extractProxyServer(outboundConfig json.RawMessage) (serverIP string, serverPort uint16, err error)` 函数：**

- unmarshal outboundConfig 为临时结构体 `struct { Server string; ServerPort int }`（json tags: `server`, `server_port`）
- 如果 `Server` 为空或 `ServerPort` 为 0，返回错误
- 如果 `Server` 是域名（不是 IP），用 `net.ResolveIPAddr("ip4", server)` 解析为 IP
- 返回 IP 字符串和端口

导入：`encoding/json`, `fmt`, `net`

```dockerfile
ARG SINGBOX_VERSION=1.13.3
RUN curl -fsSL "https://github.com/SagerNet/sing-box/releases/download/v${SINGBOX_VERSION}/sing-box-${SINGBOX_VERSION}-linux-amd64.tar.gz" \
    -o /tmp/sing-box.tar.gz \
    && tar -xzf /tmp/sing-box.tar.gz -C /tmp \
    && install -m 0755 /tmp/sing-box-${SINGBOX_VERSION}-linux-amd64/sing-box /usr/local/bin/sing-box \
    && rm -rf /tmp/sing-box* \
    && mkdir -p /etc/sing-box
```

关键要点（per D-19, D-20, D-21）：
- 版本通过 `ARG SINGBOX_VERSION=1.13.3` 管理，便于升级
- 从 GitHub Release 下载 Linux amd64 预编译二进制
- 安装到 `/usr/local/bin/sing-box`（与 nsenter 启动路径一致）
- 创建 `/etc/sing-box/` 配置目录（运行时由 SingBoxProvider 写入 config.json）
- 清理临时文件减小镜像体积
