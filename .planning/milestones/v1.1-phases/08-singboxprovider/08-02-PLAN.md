**`ApplyProxyFirewallRules` 函数：**

```go
func ApplyProxyFirewallRules(containerNS netns.NsHandle, tunIfIndex, loIfIndex, mgmtIfIndex int, proxyServerIP net.IP, proxyServerPort uint16) error
```

实现逻辑：
1. `runtime.LockOSThread()` / `defer runtime.UnlockOSThread()`
2. `nftables.New(nftables.WithNetNSFd(int(containerNS)))` 创建 nftables 连接
3. 调用 `applyProxyIPv4Rules(conn, tunIfIndex, loIfIndex, mgmtIfIndex, proxyServerIP, proxyServerPort)`
4. 调用 `applyIPv6Rules(conn, loIfIndex)` — 直接复用 firewall.go 的函数
5. `conn.Flush()` 应用规则
6. 错误返回 `&NetworkError{Type: ErrTunnelSetupFailed, ...}`

**`applyProxyIPv4Rules` 私有函数：**

创建 IPv4 filter 表，规则如下：

**OUTPUT chain（policy DROP）：**
1. lo → ACCEPT（复用 `addOifAcceptRule`）
2. tun0 → ACCEPT（复用 `addOifAcceptRule`，传入 `tunIfIndex`）
3. **mgmt0 + dst=proxyServerIP + dport=proxyServerPort → ACCEPT（TCP）**（新增）
4. **mgmt0 + dst=proxyServerIP + dport=proxyServerPort → ACCEPT（UDP）**（新增）
5. mgmt0 + ct state established,related → ACCEPT（复用 `addOifCtEstablishedRule`）

**INPUT chain（policy DROP）：**
1. lo → ACCEPT（复用 `addIifAcceptRule`）
2. tun0 → ACCEPT（复用 `addIifAcceptRule`，传入 `tunIfIndex`）
3. mgmt0 + tcp dport 22 → ACCEPT（复用 `addIifTCPDportAcceptRule`，SSH 管理）
4. mgmt0 + ct state established,related → ACCEPT（复用 `addIifCtEstablishedRule`）

**新增 helper 函数 `addOifDstPortAcceptRule`：**

```go
func addOifDstPortAcceptRule(conn *nftables.Conn, table *nftables.Table, chain *nftables.Chain, ifIndex int, dstIP net.IP, dport uint16, proto byte) {
```

匹配条件：
- `expr.Meta{Key: expr.MetaKeyOIF}` == ifIndex
- `expr.Meta{Key: expr.MetaKeyL4PROTO}` == proto（6=TCP 或 17=UDP）
- `expr.Payload{Base: expr.PayloadBaseNetworkHeader, Offset: 16, Len: 4}` == dstIP（IPv4 dst addr）
- `expr.Payload{Base: expr.PayloadBaseTransportHeader, Offset: 2, Len: 2}` == dport
- `expr.Verdict{Kind: expr.VerdictAccept}`

常量 `ipprotoUDP = 17` 在此文件中定义（`ipprotoTCP = 6` 已在 firewall.go 中）。

```go
func ensureIPForwarding(ctx context.Context) error {
    cmd := exec.CommandContext(ctx, "sysctl", "-w", "net.ipv4.ip_forward=1")
    if out, err := cmd.CombinedOutput(); err != nil {
        return fmt.Errorf("enable ip forwarding: %w (%s)", err, strings.TrimSpace(string(out)))
    }
    return nil
}
```

**2. `ensureHostMasquerade(ctx context.Context) error` 函数（幂等）：**

```go
func ensureHostMasquerade(ctx context.Context) error {
    // 检查规则是否已存在（幂等）
    check := exec.CommandContext(ctx, "iptables", "-t", "nat", "-C", "POSTROUTING",
        "-s", "10.99.0.0/16", "-j", "MASQUERADE")
    if check.Run() == nil {
        return nil // 规则已存在
    }
    // 添加 masquerade 规则：管理 veth 子网出站 NAT
    add := exec.CommandContext(ctx, "iptables", "-t", "nat", "-A", "POSTROUTING",
        "-s", "10.99.0.0/16", "-j", "MASQUERADE")
    if out, err := add.CombinedOutput(); err != nil {
        return fmt.Errorf("add masquerade rule: %w (%s)", err, strings.TrimSpace(string(out)))
    }
    return nil
}
```

关键设计点：
- masquerade 源地址 `10.99.0.0/16` 与 `InjectManagementVeth` 中分配的管理 veth 子网一致（namespace.go 行 65-66: `10.99.%d.%d/30`）
- 使用 iptables（不用 nftables on host side）避免与 Docker Engine 28.x 的 iptables 规则冲突（per RESEARCH.md Alternatives Considered）
- `ensureHostMasquerade` 是幂等的：先用 `-C` 检查是否存在，存在则跳过
- 两个函数都是 package-private（小写），由 SingBoxProvider.PrepareHost 调用

导入：`context`, `fmt`, `os/exec`, `strings`
