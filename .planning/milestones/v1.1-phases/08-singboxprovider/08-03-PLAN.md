**PrepareHost 方法（15 步流水线，与 TunnelProvider 高度对称）：**

```go
func (sp *SingBoxProvider) PrepareHost(ctx context.Context, spec HostNetworkSpec) error
```

步骤（每步失败立即返回 `&NetworkError{Type: ErrTunnelSetupFailed, HostID: spec.HostID, ...}`）：

1. **Egress 非空校验：** `spec.Egress == nil` → 返回 `ErrBindingMissing`
2. **Proxy config 非空校验：** `spec.Egress.Proxy == nil` → 返回 `ErrTunnelSetupFailed`，message: `"SingBoxProvider requires proxy config"`
3. **获取容器 netns + PID：** `GetContainerNetNS(fmt.Sprintf("cloudproxy-%s", spec.HostID))`，将 pid 赋值给 `spec.ContainerPID`
4. **获取宿主机 netns：** `netns.Get()` — defer close
5. **注入管理 veth：** `InjectManagementVeth(hostNS, containerNS, spec.HostID)` — 日志记录 hostIP/containerIP
6. **提取 proxy server 地址：** `extractProxyServer(spec.Egress.Proxy.OutboundConfig)` — 得到 serverIP/serverPort
7. **宿主机 IP 转发：** `ensureIPForwarding(ctx)`
8. **宿主机 masquerade：** `ensureHostMasquerade(ctx)`
9. **容器 netns 内添加 proxy server host route：** 调用新的私有函数 `addProxyServerRoute(containerNS, hostNS, serverIP, containerIP)`（见下方），其中 `containerIP` 即 InjectManagementVeth 返回的 hostIP（因为管理 veth 对端的网关是宿主机端 IP）。注意：这里的网关是 host 侧的 mgmt veth IP（即 `hostIP` 返回值），不是 containerIP。
10. **生成 sing-box 配置：** `buildSingBoxConfig(*spec.Egress.Proxy)` — 得到 configJSON
11. **写入容器文件系统：** `writeSingBoxConfig(spec.ContainerPID, configJSON)` — 写入 `/proc/<pid>/root/etc/sing-box/config.json`
12. **nsenter 启动 sing-box：** `startSingBox(ctx, spec.ContainerPID)` — 后台启动
13. **等待 tun0 就绪：** `waitForTun0(ctx, spec.ContainerPID, 10*time.Second)` — 轮询 200ms 间隔
14. **配置容器 DNS：** `ConfigureContainerDNS(spec.ContainerPID, spec.Egress.Proxy.DNSServer)`
15. **解析容器接口 ifindex：** `resolveProxyIfIndexes(containerNS, hostNS)` — 新函数，查找 tun0/lo/mgmt0
16. **应用防火墙规则：** `ApplyProxyFirewallRules(containerNS, tunIdx, loIdx, mgmtIdx, net.ParseIP(serverIP), serverPort)`
17. **三重校验：** `VerifyNetworkIntegrity(ctx, spec.ContainerPID, *spec.Egress)` — 日志记录结果

**私有 helper 函数：**

**`addProxyServerRoute(containerNS, hostNS netns.NsHandle, proxyServerIP string, mgmtGateway string) error`：**
- `runtime.LockOSThread()` / defer
- `netns.Set(containerNS)` / defer `netns.Set(hostNS)`
- `netlink.LinkByName("mgmt0")` 获取 mgmt link
- 构建 route: `Dst: proxyServerIP/32, Gw: mgmtGateway, LinkIndex: mgmt0`
- `netlink.RouteAdd(route)`

**`writeSingBoxConfig(containerPID uint32, config []byte) error`：**
- 路径: `filepath.Join("/proc", fmt.Sprintf("%d", containerPID), "root", "etc", "sing-box", "config.json")`
- `os.WriteFile(path, config, 0o644)`
- 错误包装为 `NetworkError`

**`startSingBox(ctx context.Context, containerPID uint32) error`：**
- `exec.CommandContext(ctx, "nsenter", "-t", pidStr, "-n", "-m", "-p", "--", "/usr/local/bin/sing-box", "run", "-c", "/etc/sing-box/config.json")`
- `cmd.Start()` — 不调用 `cmd.Wait()`
- `go cmd.Wait()` 防止僵尸进程
- 日志记录 sing-box 进程已启动

**`waitForTun0(ctx context.Context, containerPID uint32, timeout time.Duration) error`：**
- 使用 `time.After(timeout)` 和 `time.NewTicker(200 * time.Millisecond)`
- 每次 tick 执行 `nsenter -t <pid> -n -- ip link show tun0`
- 成功（exit 0）返回 nil，超时返回错误

**`resolveProxyIfIndexes(containerNS, hostNS netns.NsHandle) (tunIdx, loIdx, mgmtIdx int, err error)`：**
- 与 `resolveContainerIfIndexes` 结构相同，但查找 `"tun0"` 替代 `wgIfName`
- `runtime.LockOSThread()` / `netns.Set(containerNS)` / defer `netns.Set(hostNS)`
- `netlink.LinkByName("tun0")` / `netlink.LinkByName("lo")` / `netlink.LinkByName("mgmt0")`

**CleanupHost 方法：**


3. 返回 nil

**PrepareHost 方法（per D-01, D-02）：**

```go
func (rp *RoutingProvider) PrepareHost(ctx context.Context, spec HostNetworkSpec) error {
    if spec.Egress == nil {
        return &NetworkError{
            Type:    ErrBindingMissing,
            Message: "PrepareHost called without egress config",
            HostID:  spec.HostID,
        }
    }
    switch spec.Egress.TunnelType {
    case TunnelTypeProxy:
        rp.logger.Info("routing to sing-box provider", "host_id", spec.HostID, "tunnel_type", spec.Egress.TunnelType)
        return rp.singbox.PrepareHost(ctx, spec)
    default:
        rp.logger.Info("routing to wireguard provider", "host_id", spec.HostID, "tunnel_type", spec.Egress.TunnelType)
        return rp.tunnel.PrepareHost(ctx, spec)
    }
}
```

**CleanupHost 方法（per D-02 — 两种类型都尝试清理）：**

```go
func (rp *RoutingProvider) CleanupHost(ctx context.Context, spec HostNetworkSpec) error {
    rp.tunnel.CleanupHost(ctx, spec)
    rp.singbox.CleanupHost(ctx, spec)
    return nil
}
```
