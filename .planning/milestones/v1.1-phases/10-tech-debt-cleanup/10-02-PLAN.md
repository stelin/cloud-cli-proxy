**变更 1：在文件顶部（`EgressIPsPage` 函数之前）添加两个工具函数：**

```typescript
const TEST_RESULTS_KEY = "egress-ip-test-results";

function loadTestResults(): Map<string, TestResult> {
  try {
    const raw = localStorage.getItem(TEST_RESULTS_KEY);
    if (raw) return new Map(JSON.parse(raw));
  } catch {
    // corrupt data — start fresh
  }
  return new Map();
}

function saveTestResults(results: Map<string, TestResult>) {
  try {
    localStorage.setItem(TEST_RESULTS_KEY, JSON.stringify([...results.entries()]));
  } catch {
    // quota exceeded — ignore
  }
}
```

放在 `formatDate` 函数之后、`EgressIPsPage` 函数之前。

**变更 2：修改 `testResults` 的初始值：**

当前（第 78-80 行）：
```typescript
const [testResults, setTestResults] = useState<Map<string, TestResult>>(
  new Map(),
);
```

修改为：
```typescript
const [testResults, setTestResults] = useState<Map<string, TestResult>>(loadTestResults);
```

注意：`useState(loadTestResults)` 传入函数引用（lazy initializer），不是 `useState(loadTestResults())`。

**变更 3：修改 `handleTest` 函数（第 88-101 行）：**

当前：
```typescript
function handleTest(ip: EgressIP) {
  setTestingId(ip.id);
  testMutation.mutate(ip.id, {
    onSuccess: (result) => {
      setTestResults((prev) => new Map(prev).set(ip.id, result));
      setTestDialogResult(result);
      setTestingId(null);
    },
    onError: () => {
      toast.error("测试失败");
      setTestingId(null);
    },
  });
}
```

修改为：
```typescript
function handleTest(ip: EgressIP) {
  if (ip.tunnel_type !== "proxy") {
    toast.info("WireGuard 类型出口 IP 在容器启动时自动验证，不支持手动测试");
    return;
  }
  setTestingId(ip.id);
  testMutation.mutate(ip.id, {
    onSuccess: (result) => {
      setTestResults((prev) => {
        const next = new Map(prev).set(ip.id, result);
        saveTestResults(next);
        return next;
      });
      setTestDialogResult(result);
      setTestingId(null);
    },
    onError: () => {
      toast.error("测试失败");
      setTestingId(null);
    },
  });
}
```

两处变更：
1. 函数入口添加 `tunnel_type !== "proxy"` 前置判断，命中时 `toast.info(...)` 后 return
2. `onSuccess` 中 `setTestResults` 回调内在 set 后调用 `saveTestResults(next)` 写入 localStorage

**变更 4：测试菜单项禁用 WireGuard 类型（约第 241-251 行）：**

当前：
```tsx
<DropdownMenuItem
  onClick={() => handleTest(ip)}
  disabled={testingId === ip.id}
```

修改为：
```tsx
<DropdownMenuItem
```

这样 WireGuard 类型的测试菜单项会显示为禁用状态（灰色文字），配合 `handleTest` 中的 toast 提示形成双重保护。
