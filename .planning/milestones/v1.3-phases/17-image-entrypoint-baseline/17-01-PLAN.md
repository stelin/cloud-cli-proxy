创建 `claude-shell/docker/Dockerfile`，内容按以下顺序组织：

**1. 基础镜像和环境变量：**
```dockerfile
FROM ubuntu:24.04
ENV DEBIAN_FRONTEND=noninteractive
```

**2. 系统工具安装（per D-02，最小必要集）：**
```dockerfile
RUN apt-get update \
    && apt-get install -y --no-install-recommends \
        curl git jq procps iproute2 ca-certificates bash sudo nftables \
    && rm -rf /var/lib/apt/lists/*
```

**3. sing-box 二进制安装（per D-03，复用 sing-box-gateway 模式）：**
```dockerfile
ARG SINGBOX_VERSION=1.13.3
RUN set -eux; \
    ARCH="$(dpkg --print-architecture)"; \
    curl -fsSL -o /tmp/sb.tgz \
      "https://github.com/SagerNet/sing-box/releases/download/v${SINGBOX_VERSION}/sing-box-${SINGBOX_VERSION}-linux-${ARCH}.tar.gz"; \
    tar -xzf /tmp/sb.tgz -C /tmp; \
    install -m 0755 "/tmp/sing-box-${SINGBOX_VERSION}-linux-${ARCH}/sing-box" /usr/local/bin/sing-box; \
    rm -rf /tmp/sb.tgz "/tmp/sing-box-${SINGBOX_VERSION}-linux-${ARCH}"; \
    sing-box version
```

**4. 容器工作用户创建：**
用户名 `claude`（UID 1000, GID 1000），HOME 为 `/home/claude`，sudoers NOPASSWD：
```dockerfile
ARG CLAUDE_USER=claude
ARG CLAUDE_UID=1000
ARG CLAUDE_GID=1000

RUN groupadd --gid "${CLAUDE_GID}" "${CLAUDE_USER}" \
    && useradd --uid "${CLAUDE_UID}" --gid "${CLAUDE_GID}" \
       --home-dir "/home/${CLAUDE_USER}" --create-home \
       --shell /bin/bash "${CLAUDE_USER}" \
    && echo "${CLAUDE_USER} ALL=(ALL) NOPASSWD:ALL" \
       > "/etc/sudoers.d/${CLAUDE_USER}" \
    && chmod 0440 "/etc/sudoers.d/${CLAUDE_USER}"
```

**5. Claude Code 安装（per D-04、D-05）：**
必须先 `USER claude` 再执行安装脚本，安装路径绑定 `$HOME`：
```dockerfile
USER ${CLAUDE_USER}
RUN curl -fsSL https://claude.ai/install.sh | bash
```

**6. PATH 和环境变量（per D-06）：**
```dockerfile
ENV PATH="/home/claude/.local/bin:${PATH}"
ENV DISABLE_AUTOUPDATER=1
```

**7. entrypoint 安装（需要 root 权限 COPY 到 /usr/local/bin）：**
```dockerfile
USER root
COPY entrypoint.sh /usr/local/bin/entrypoint.sh
RUN chmod +x /usr/local/bin/entrypoint.sh
```

**8. 最终设置：**
```dockerfile
USER ${CLAUDE_USER}
WORKDIR /home/claude
ENTRYPOINT ["/usr/local/bin/entrypoint.sh"]
```

注意使用 exec form `ENTRYPOINT [...]`，不要用 shell form（会导致 shell 成为 PID 1，信号转发异常）。

**层优化要点：** 将 apt-get 安装合并为一个 RUN 指令；Claude Code 安装放在 entrypoint.sh COPY 之前，避免 entrypoint 修改导致 Claude Code 重新下载。

创建 `claude-shell/docker/entrypoint.sh`，内容如下：

**1. 脚本头部：**
```bash
#!/usr/bin/env bash
set -euo pipefail
```

**2. 退出码常量（per D-09）：**
```bash
readonly EXIT_NETWORK=10
readonly EXIT_FINGERPRINT=20
readonly EXIT_ANTI_DETECT=30
readonly EXIT_CLAUDE=40
```

**3. 日志工具函数：**
```bash
log() { echo "[entrypoint] $(date -u +%Y-%m-%dT%H:%M:%SZ) $*"; }
die() { log "FATAL: $1"; exit "${2:-1}"; }
```

**4. 步骤函数（per D-08）：**

`setup_network()` — Phase 17 为占位（per D-11）：
```bash
setup_network() {
  log "step=network status=placeholder"
}
```

`setup_fingerprint()` — Phase 17 为占位（per D-11）：
```bash
setup_fingerprint() {
  log "step=fingerprint status=placeholder"
}
```

`setup_anti_detect()` — Phase 17 为占位：
```bash
setup_anti_detect() {
  log "step=anti_detect status=placeholder"
}
```

`start_claude()` — 查找 claude 二进制，exec 替换为 PID 1（per D-10）：
```bash
start_claude() {
  log "step=claude status=starting"
  local claude_bin
  claude_bin="$(command -v claude 2>/dev/null)" \
    || die "claude binary not found in PATH" $EXIT_CLAUDE
  exec "$claude_bin" "$@"
}
```

**5. 主流程调用（per D-08 顺序：网络 → 指纹 → 反检测 → Claude）：**
```bash
setup_network    || die "network setup failed"    $EXIT_NETWORK
setup_fingerprint || die "fingerprint setup failed" $EXIT_FINGERPRINT
setup_anti_detect || die "anti-detect setup failed" $EXIT_ANTI_DETECT
start_claude "$@"
```

**关键注意事项：**
- `start_claude` 函数接收 `"$@"` 以透传容器启动参数给 Claude Code
- 占位函数只输出日志，不执行任何实际操作，不会触发 `set -e` 退出
- `command -v claude 2>/dev/null` 使用 `||` 短路避免 `set -e` 在找不到命令时意外退出
- `exec` 替换当前 shell 进程，Claude Code 成为 PID 1（或 tini 子进程）
