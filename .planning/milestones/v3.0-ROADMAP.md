# Milestone v3.0: 远端开发体验升级

**Status:** ✅ SHIPPED 2026-04-23
**Phases:** 29-35（含 Phase 29.1 P0 hotfix）
**Total Plans:** 30
**Goal:** 在已 ship 的 v2.0 cloud-claude 基础上做体验升级——三层文件系统（Mutagen + sshfs + mergerfs）、SSH 会话弱网容忍 + tmux 默认包装与多端共享、Claude Code OAuth 状态持久化、`cloud-claude doctor` 与统一错误码体系——把 CLI 从"能跑"升级到"能成为日常开发主战场"。零增量特权，复用 v2.0 已开放的 FUSE / SYS_ADMIN / AppArmor unconfined 通道。

## Overview

v3.0 主线分三块：

- **A · 文件映射架构重构**：Mutagen 热同步源码 + sshfs 冷兜底 + mergerfs 联合视图，单一 `/workspace` 对用户透明；`--mount-mode=auto|full|mutagen-only|sshfs-only` 四档降级状态机
- **B · 会话可靠性**：SSH KeepAlive 15s/4 + 退避重连 + BufferedStdin 灰色未确认 + tmux 默认包装 + 多端共享 attach + 账号级 Mutagen 单例锁
- **C · 运维与体验配套**：`cloud-claude doctor` 5 维度（network/auth/ssh/mount/disk）+ 6 类自动 fix + Claude Code OAuth 持久化（`claude-state-{account_id}` named volume + admin DELETE 联动 volume rm）+ 42 条 8 域错误码 + `cloud-claude explain <code>`

- **persistent volume 单 volume 命名 (Q4)**：`claude-state-{claude_account_id}` + label `com.cloud-cli-proxy.account_id`
- **Entry API 扩展字段 (Q5)**：在现有 `/v1/entry/{id}/auth` 加字段（向后兼容），不新增 `/capabilities` endpoint
- **host-agent 不扩展返回 image labels (Q6)**：保边界守恒
- **doctor 不给 host-agent 加 endpoint (ARCHITECTURE §6)**：本地 + SSH 实现，控制 / 数据平面分离
- **per-claude_account tmux session 命名 (Q8)**：与 volume 命名一致
- **doctor --fix 默认幂等 + stdin y/N + CI --yes (Q9)**：避免 destructive 操作意外执行
- **first-sync alpha/beta 安全门 (Q7)**：MOUNT_MUTAGEN_SAFETY_GUARD Fatal 防止反向清空本地仓库
