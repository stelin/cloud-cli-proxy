## Decisions Made

- **整批补齐 6 处而非只修 GetHost**：RESEARCH §1.3 覆盖矩阵明确列出 6 处同类漏列，任一 admin / reconciler 链路从 List* 读 host 后写回都会让密码再次丢失。一次 commit 补齐杜绝增量回归。
- **沿用既有 const SQL 提升模式**：`getHostByShortIDSQL`（queries.go:1163-1170）已是同仓库标准做法，命名（snake → camelCase + `SQL` 后缀）、注释风格（"将 SQL 文本提升为包级常量"）、raw string 缩进全部对齐，无引入新风格成本。
- **测试只做字符串契约、不连接 DB**：仓库无 testcontainers / dockertest 基础设施（RESEARCH §0），且本 plan scope 严控不引入新测试框架。`strings.Contains` 足以锁定回归源头。
