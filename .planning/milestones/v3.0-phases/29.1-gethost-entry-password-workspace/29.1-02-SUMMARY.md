# Phase 29.1 Plan 02: worker / runtime_service fail-fast 改造 + RecordEvent + 单测 Summary

**消灭 runtime 层最后两处 `firstNonEmpty(..., "workspace")` 形式的空密码 fallback：runtime_service.QueueHostAction、worker.buildCreateArgs、worker.syncContainerCredentials 三处入口在 EntryPassword 为空时统一改走 fail-fast + RecordEvent，让"密码丢了"从"用户登不进容器"升级为"任务明确失败 + 可查 runtime.entry_password_missing 事件"。新增两条 worker 单测锁定回归。**

## Accomplishments

### Task 2.1：runtime_service.QueueHostAction
- `Service.repo` 匿名 interface（runtime_service.go:32-39）与 `NewService` 形参 interface（46-57）各追加一行 `RecordEvent(context.Context, repository.RecordEventParams) (repository.Event, error)`，concrete `*Repository` 已实现（queries.go:763），调用方 `internal/controlplane/app/app.go:112` 零编译错误。
- 替换原 `slog.Warn("host entry_password is empty, container will use default password", ...)`（旧第 149-152 行）为 fail-fast 块（HEAD 第 151-169 行）：`request.EntryPassword == ""` → `s.repo.RecordEvent(level=error, type=runtime.entry_password_missing, metadata={host_id, action, source:"queue"})` → `slog.Error(...)` → `return repository.Task{}, fmt.Errorf("host %s entry_password is empty; refusing to queue %s", hostID, action)`。

### Task 2.2：worker.go

## Decisions Made

- **Service.repo 接口最小扩展而非引入新 EventRecorder 字段**：PATTERNS §3 已建议此模式；Service 当前匿名 interface 已聚合 4 个方法（GetHost / GetUser / CreateTask / ListSSHKeysByUser），追加第 5 个方法无破坏性、与现有风格一致；concrete `*Repository` 早已实现 `RecordEvent`，调用方 app.go 零变化。
- **PLAN 字面替换位置 vs. 提前到 CreateTask 之前**：PLAN action step 2 明确以 `slog.Warn` 行号为锚（149-152），而 CreateTask 在 97 行已先行执行。本 plan 严守 PLAN 字面，fail-fast 后 task 行残留于 pending — 接受这一已知后果（task 表已有 expiry / 清理逻辑），不重构 task 创建顺序避免扩大 scope。
- **syncContainerCredentials 改用 execInContainer**：PLAN Task 2.3 note 1 显式预批为"可测性最小重构"。改造后该函数 chpasswd 调用与 worker.go:673 包级 var 统一，单测可通过 setupInjectTest 替换 execInContainer 来捕获或抑制 chpasswd 调用流。语义等价（`bash -c "chpasswd"` 仍从 stdin 读 user:pass），不动 stdin 内容、不动 RecordEvent 失败分支。
- **minimalCreateHostRequest 工厂补 EntryPassword 占位**：PATTERNS §6 已预判，PLAN Task 2.3 note 2 也提到"若工厂被他测依赖改为了非空默认，保持 override 语义即可"。占位字符串 `"pw-default-test"` 不是真实密码、明显的测试标记，T-29.1-05-log 不受影响。
- **RecordEvent metadata 严守 T-29.1-05-log**：runtime_service.go fail-fast 事件 metadata 字段为 `host_id`/`action`/`source`，syncContainerCredentials fail-fast 事件 metadata 字段为 `host_id`/`container`/`source`，全部为业务结构化字段，绝不含 `entry_password`/`password`/`pass` 字面 key 或对应 EntryPassword 字符串值；Message 是固定英文短句，不插值密码。
