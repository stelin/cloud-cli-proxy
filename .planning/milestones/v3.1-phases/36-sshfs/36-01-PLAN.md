// ── Phase 36 新增 ──────────────────────────────────────────────────────────
registerExplanation(MOUNT_REQUIRE_GIT_REPO, `触发场景：用户在非 git 仓库目录（如 /tmp、家目录）运行 cloud-claude，系统执行 git rev-parse --show-toplevel 返回非 0，判定当前 cwd 不在任何 git 仓库内。

根本原因：cloud-claude 的热同步（hot_sync）机制需要以 git 仓库为边界界定同步范围；若不加约束，热同步可能误上传整个家目录或 /tmp 下的所有文件，造成大量不必要的文件传输，并有覆盖远端工作目录的风险（PITFALLS C5）。git 仓库存在意味着用户明确划定了项目边界，与 cloud-claude 的"同步你的项目，而不是你的家目录"定位完全一致。

复现方式：cd /tmp && cloud-claude —— /tmp 是临时目录，不含 .git，立即触发此错误，exit 4（exitConfigError）；或 mkdir /tmp/test && cd /tmp/test && cloud-claude。

修复路径：（1）cd 到已有 git 仓库目录后重新运行 cloud-claude，如 cd ~/projects/my-repo && cloud-claude；（2）在当前目录执行 git init 初始化仓库后重试；（3）--mount-mode 任何档位都无法绕过此检查——这是全局前置闸门，与挂载模式正交。

关联文档：REQUIREMENTS.md §REQ-MOUNT-V31-01；31-CONTEXT.md §D-11（整目录级熔断，与本检查互补：整目录 >50MB 拒绝热同步，git 检查拒绝非项目目录入口）。`)

registerExplanation(MOUNT_OVERSIZED_FILE_SKIPPED, `触发场景：热同步初始扫描阶段发现某个文件（未被 .gitignore 或默认二进制黑名单命中）的大小超过了 hot_sync_max_file_mb 配置阈值（默认 50MB）。该文件被跳过热同步，由 cold sshfs 分支提供读取访问，同时写入 last-session.json::oversized_files 列表。

根本原因：大文件（如视频、训练数据集、编译产物）进入热同步会显著拉长初始同步耗时，且这类文件通常不需要实时双向同步；cold sshfs 兜底保证文件可读，配合 Phase 36 新增的 sshfs 内核页面缓存（cache=yes,kernel_cache），同会话内第二次读取无需回源。

复现方式：在 git 仓库中放入一个 60MB 的 model.bin 文件（不在 .gitignore 中），运行 cloud-claude，观察启动时 stderr 的 [!] 跳过大文件提示以及 ~/.cloud-claude/last-session.json::oversized_files 数组。

修复路径：（1）编辑 ~/.cloud-claude/config.yaml 调大 hot_sync_max_file_mb（如 200），让更大的文件也进入热同步；（2）在 .gitignore 或 ~/.cloud-claude/ignore 中加入该文件的 glob，使其被 ignore 过滤而不触发此警告；（3）接受现状——文件通过 cold sshfs + 内核页面缓存访问，功能完整，首次读后缓存命中，体验接近本地。

关联文档：REQUIREMENTS.md §REQ-MOUNT-V31-02 / REQ-MOUNT-V31-03；31-CONTEXT.md §D-11（整目录级 >50MB 拒绝，与单文件熔断互补：整目录保护巨型 mono-repo，单文件保护正常仓库内的超大资源）。`)
