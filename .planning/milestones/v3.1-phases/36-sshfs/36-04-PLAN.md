**Step 1 — 新建 cmd/cloud-claude/git_check.go（package main）：**

```go
package main

import (
	"fmt"
	"os/exec"

	"github.com/zaneliu/cloud-cli-proxy/internal/cloudclaude/errcodes"
)

// requireGitRepo 在 cwd 执行 git rev-parse --show-toplevel。
// 非 git 仓库（含 git 未安装、git worktree、submodule、detached HEAD 均视为 git 仓库通过）
// 返回包含 MOUNT_REQUIRE_GIT_REPO 格式化错误消息的 error。
// 退出码 0 = git 仓库；非 0 = 非 git 仓库或 git 不可用（按同一错误处理，D-03）。
func requireGitRepo(cwd string) error {
	cmd := exec.Command("git", "rev-parse", "--show-toplevel")
	cmd.Dir = cwd
	if err := cmd.Run(); err != nil {
		return fmt.Errorf("%s", errcodes.Format(errcodes.MOUNT_REQUIRE_GIT_REPO, cwd))
	}
	return nil
}
```

注意：import path 需与项目 go.mod 中的 module path 一致（请读取 go.mod 第一行确认）。

// TestRequireGitRepo_InGitRepo 使用当前 workspace（cloud-cli-proxy 本身是 git 仓库）
func TestRequireGitRepo_InGitRepo(t *testing.T) {
	cwd, err := os.Getwd()

	if err := requireGitRepo(cwd); err != nil {

// TestRequireGitRepo_NotAGitRepo 使用临时空目录
func TestRequireGitRepo_NotAGitRepo(t *testing.T) {
	dir := t.TempDir() // 临时空目录，不是 git 仓库
	err := requireGitRepo(dir)
	if err == nil {
		t.Fatal("非 git 仓库目录应返回 error，got nil")

	if !strings.Contains(err.Error(), "MOUNT_REQUIRE_GIT_REPO") {
		t.Errorf("error 消息应含 MOUNT_REQUIRE_GIT_REPO，got: %v", err)

// TestRequireGitRepo_GitUnavailableHandled 验证 git 不可用按非 git 仓库处理

func TestRequireGitRepo_GitUnavailableHandled(t *testing.T) {
	// 直接用临时目录 + 系统 git 未安装的降级路径

	if err == nil {

	if !strings.Contains(err.Error(), "MOUNT_REQUIRE_GIT_REPO") {

## Trust Boundaries

| 边界 | 说明 |
|------|------|
| os.Getwd() → requireGitRepo | cwd 来自 OS，不受用户输入污染；git 命令路径固定 |
| exec.Command("git", ...) | 参数为固定字面量（rev-parse --show-toplevel），cwd 来自 os.Getwd()，无注入风险 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-04-01 | Spoofing | git rev-parse 结果 | accept | git 二进制完整性属于操作系统安全边界，超出 cloud-claude 防御范围 |
| T-36-04-02 | Tampering | PATH 劫持导致伪造 git 命令 | accept | 与 T-36-04-01 同层；用户持有 PATH 控制权，攻击者已有本地访问权 |
| T-36-04-03 | Elevation of Privilege | exitInternalError(5) 路径 | accept | os.Getwd() 失败是极罕见系统错误，快速 exit 5 是最安全处理方式 |
| T-36-04-04 | Denial of Service | git 命令挂起 | mitigate | exec.Command 在本地文件系统执行，`git rev-parse` 无网络调用，通常 <100ms；若需 timeout 可用 exec.CommandContext（executor 可按实际情况决定是否加 3s timeout） |
