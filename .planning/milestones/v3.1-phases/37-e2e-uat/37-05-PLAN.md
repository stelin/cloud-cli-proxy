创建 `tests/scripts/uat-v31-promotion.sh`（约 500-600 行），按以下结构组织：

**框架层：**
- `#!/usr/bin/env bash` + `set -euo pipefail`
- pass/fail/skip/info helper 函数 + 计数器
- `usage()` 帮助文本
- 参数解析：`--dry-run`（默认 true）、`--confirm-destructive`、`--output-dir`、`--help`
- `trap cleanup EXIT INT TERM`（清理临时 fixture 目录）
- JSON 报告输出函数 `write_json_report`

**场景 1 — 非 git 目录拒绝挂载（scenario_git_reject）：**
- `mktemp -d` 创建非 git 目录
- 尝试 `cloud-claude --mount-mode=full`（或通过 dry-run 模拟）
- 断言：stderr 含 `MOUNT_REQUIRE_GIT_REPO` 或退出码为 exitConfigError

**场景 2 — 大文件熔断（scenario_oversized_skip）：**
- 创建 fixture git 仓库，含 1 个 60MB 文件（`dd if=/dev/zero of=big.bin bs=1M count=60`）
- `--dry-run` 时仅打印 fixture 构造命令
- `--confirm-destructive` 时执行 mount，断言：
  - stderr 含 "跳过大文件"
  - `last-session.json` 的 `oversized_files` 数组非空
  - hot 分支不含 big.bin

**场景 3 — FUSE cache 命中（scenario_fuse_cache_hit）：**
- 创建 fixture PNG（1MB 随机数据）
- mount → `cat fixture.png` → 记录 SFTP read count
- 30s 内再次 `cat fixture.png` → SFTP read count 不变
- 断言：FUSE page cache 生效

**场景 4 — 冷文件晋升（scenario_cold_promotion）：**
- 创建 fixture PNG（1MB 随机数据），确保不在 .gitignore 中但被默认黑名单命中（进入 cold 分支）
- mount → `cat fixture.png` → sleep 6s → 检查 hot 分支是否出现该文件
- 再次 `cat fixture.png` → SFTP read count 不变
- 断言：晋升成功 + mergerfs 热命中（REQ-MOUNT-V31-11）

**场景 5 — CLOUD_CLAUDE_NO_PROMOTION=1（scenario_no_promotion）：**
- 设置环境变量后 mount
- 断言：
  - PID file 不存在
  - `last-session.json` 的 `promotion_count` 为 0 或 null
  - watcher 未启动

**场景 6 — JSON 报告格式（scenario_json_report）：**
- 运行后生成 JSON 报告
- 断言：`schema_version == 1`、`scenarios` 数组非空
- 每个 scenario 有 `name/status/assertions` 字段

**--dry-run 行为：**
- 打印每个场景将执行的操作描述
- 不创建真实 fixture、不启动 mount
- 退出码 0（dry-run 本身视为 PASS）

**--confirm-destructive 行为：**
- 创建 fixture → mount → 断言 → cleanup → 下个场景
- 所有场景跑完输出 JSON 报告
- 任一场景 FAIL → 退出码 1；部分 SKIP + 其余 PASS → 退出码 0

**JSON 报告格式：**
```json
{
  "schema_version": 1,
  "script": "uat-v31-promotion.sh",
  "timestamp": "...",
  "dry_run": false,
  "summary": {"pass": N, "fail": N, "skip": N},
  "scenarios": [
    {
      "name": "git_reject",
      "status": "pass",
      "assertions": [{"name": "exit code", "result": "pass"}, ...]
    }
  ]
}
```
