## Milestone Summary

**Key Decisions:**

- SSH Proxy channel dispatch 使用 switch-case 路由（session / direct-tcpip / reject）
- `handleGlobalRequests` 使用 `ssh.Conn` 接口保持通用性
- `handleConnection` 预 dial 共享 targetClient 避免 per-channel dial 开销
- `isForbiddenTarget` 设计为纯函数，不依赖 Server 结构体
- entrypoint `MODE=local` 跳过 KasmVNC + control-plane 心跳，保留 sshd + sing-box
- 本地版 sing-box 以 sidecar 容器运行，复用 Cloud 版网络模型
- 本地版默认深度伪装，不区分 stealth/local 模式
- Phase 42/43/44 gap closure 链：第一轮审计发现 3 个关键缺口 → 三阶段全关闭

**Issues Resolved:**

- VS Code Remote-SSH 需要 direct-tcpip channel → Phase 38 实现完整转发
- 本地容器需独立于 control-plane → Phase 39 实现 cloud-claude local 子命令
- 审计发现 Phase 39 缺少 VERIFICATION.md → Phase 42 补齐
- UAT 脚本缺少端口转发场景 → Phase 43 补充 3 个场景
- doctor 不主动验证 sshd_config → Phase 44 新增 parseSSHDForwarding

**Technical Debt:**

- 11 项人工验证场景待 Docker 环境执行（Phase 38 x3, Phase 39 x5, Phase 43 x3）
- Phase 42/44 缺少独立 VERIFICATION.md（产出物在其他文件中，可接受）
