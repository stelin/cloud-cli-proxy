创建 `tests/scripts/uat-vscode-remote-ssh.sh`，覆盖以下场景：

**CLI 参数：**
- `--dry-run`（默认）：打印每个场景的操作描述，不做实际断言
- `--confirm-destructive`：执行实际断言（需要运行中的容器）
- `--container=NAME`：指定容器名（默认自动检测 `cloud-claude-local-*`）
- `--expected-egress-ip=IP`：指定期望的出口 IP（可选，不指定则跳过出口 IP 断言）
- `--help, -h`：显示帮助

**场景覆盖（6 个场景）：**

1. **sing-box 进程检测**：`docker exec $CONTAINER pgrep -x sing-box` 确认进程存在
2. **出口 IP 验证**：`docker exec $CONTAINER curl -s --max-time 15 ifconfig.me` 返回值等于 `--expected-egress-ip`
3. **DNS 泄漏检测**：`docker exec $CONTAINER nslookup ifconfig.me` 成功解析（走 sing-box DNS）
4. **VS Code Server 进程检测**：`docker exec $CONTAINER pgrep -f vscode-server`（可选，可能未连接）
5. **sshd 进程检测**：`docker exec $CONTAINER pgrep -x sshd` 确认 sshd 运行
6. **sing-box 日志域名检查**：检查 sing-box 日志中是否有 `update.code.visualstudio.com` 或 `marketplace.visualstudio.com`

**JSON 报告格式（与 uat-v31-promotion.sh 一致）：**
```json
{
  "schema_version": 1,
  "script": "uat-vscode-remote-ssh.sh",
  "timestamp": "2026-05-08T...",
  "dry_run": true/false,
  "summary": {"pass": N, "fail": N, "skip": N},
  "outcome": "pass|fail|skip",
  "scenarios": [
    {"name": "singbox_process", "status": "pass|fail|skip", "assertions": [...]}
  ]
}
```

**退出码：** 0=PASS, 1=FAIL, 2=SKIP

**脚本风格：** 与 `tests/scripts/uat-v31-promotion.sh` 完全一致（pass/fail/skip/info 函数、JSON 报告、dry-run 模式、清理 trap）。

**自动检测容器名：**
```bash
detect_container() {
  local name
  name=$(docker ps --filter "label=cloud-claude-local=true" --format '{{.Names}}' | head -1)
  if [ -z "$name" ]; then
    name=$(docker ps --filter "name=cloud-claude-local" --format '{{.Names}}' | head -1)
  fi
  echo "$name"
}
```

**关键注意点：**
- macOS 上 sing-box 可能以 proxy 模式运行（无 tun），场景 1 仍适用（进程检测不区分模式）
- 出口 IP 验证需要 `--expected-egress-ip` 参数，未指定时 SKIP
- VS Code Server 检测可能 SKIP（用户可能尚未通过 VS Code 连接）
- sing-box 日志路径：先尝试 `/var/log/sing-box.log`，再尝试 `journalctl -u sing-box`，最后尝试 `docker logs`

创建 `.planning/phases/40-vs-code-remote-ssh-e2e/40-MANUAL-CHECKLIST.md`，包含以下内容：

**前置条件：**
- 宿主机已安装 VS Code + Remote-SSH 扩展
- 已运行 `cloud-claude local --egress-config <file>` 启动容器
- 容器已启动且 sing-box 运行中

**Happy Path 测试（5 步）：**

1. **VS Code 连接**
   - 操作：VS Code → Remote-SSH → 连接 `user@127.0.0.1:2222`
   - 预期：VS Code 成功连接，显示远程文件系统
   - 验证点：无超时、无认证失败

2. **文件浏览**
   - 操作：在 VS Code 资源管理器中浏览 `/workspace` 目录
   - 预期：文件列表正常显示，可打开文件
   - 验证点：文件内容正确加载

3. **终端操作**
   - 操作：VS Code → 终端 → 运行 `claude --version` 和 `whoami`
   - 预期：`claude` 命令可用，`whoami` 返回容器用户
   - 验证点：终端交互正常，无卡顿

4. **端口转发**
   - 操作：在容器内运行 `python3 -m http.server 8888`，VS Code 自动检测端口转发
   - 预期：VS Code 提示端口 8888 已转发，宿主机可访问 `http://localhost:8888`
   - 验证点：HTTP 请求成功返回文件列表

5. **扩展安装**
   - 操作：VS Code → 扩展 → 搜索并安装一个轻量扩展（如 "Go" 或 "Python"）
   - 预期：扩展下载并安装成功
   - 验证点：扩展在远程侧可用

**Egress 强约束测试（4 步）：**

6. **出口 IP 验证**
   - 操作：终端运行 `curl -s ifconfig.me`
   - 预期：返回的 IP 等于 egress 配置的 ExpectedIP
   - 验证点：不是宿主机出口 IP

7. **DNS 泄漏验证**
   - 操作：终端运行 `nslookup ifconfig.me` 和 `nslookup google.com`
   - 预期：DNS 解析成功，走 sing-box DNS
   - 验证点：不返回宿主机 DNS 服务器地址

8. **VS Code 更新流量验证**
   - 操作：触发 VS Code Server 更新（或检查已有的 sing-box 日志）
   - 预期：sing-box 日志中出现 `update.code.visualstudio.com` 走 proxy-out
   - 验证点：不走直连

9. **端口转发出口 IP 验证**
   - 操作：在容器内运行 `python3 -c "import http.server; ..."` 返回请求来源 IP，通过 VS Code 端口转发从宿主机 curl 访问
   - 预期：请求来源 IP 是容器内部 IP（非宿主机 IP）
   - 验证点：流量走 tunnel 而非直连

**Checklist 格式：**
每个步骤包含：操作描述、预期结果、实际结果（空白待填）、PASS/FAIL（空白待填）。
