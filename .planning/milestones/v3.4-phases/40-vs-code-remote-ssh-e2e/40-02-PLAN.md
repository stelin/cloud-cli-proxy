UAT 脚本使用方式:
```bash
# dry-run 模式（不需要容器）
bash tests/scripts/uat-vscode-remote-ssh.sh --dry-run
```

cloud-claude local 启动容器:
```bash
# 启动容器（无 egress）
cloud-claude local up

# 输出: SSH 连接信息 (host, port, user, password)
```

SSH 连接信息格式:
```
SSH 连接信息:
  Host: 127.0.0.1
  Port: XXXXX
  User: workspace
  Password: XXXXXXXX

```

**步骤 1: VS Code 连接（Happy Path 步骤 1）**

1. 打开 VS Code
2. 按 `Ctrl+Shift+P` → "Remote-SSH: Connect to Host..."
3. 输入 `workspace@127.0.0.1:PORT`（使用 Task 1 记录的端口）
4. 输入密码（使用 Task 1 记录的密码）
5. 等待 VS Code 连接成功

**预期：** VS Code 成功连接，显示远程文件系统。
**记录：** 连接是否成功，耗时，是否有错误提示。

**步骤 2: 文件浏览（Happy Path 步骤 2）**

1. 在 VS Code 资源管理器中浏览 `/workspace` 目录
2. 打开一个文件查看内容

**预期：** 文件列表正常显示，文件内容正确加载。

**步骤 3: 终端操作（Happy Path 步骤 3）**

1. VS Code → 终端（`` Ctrl+` ``）
2. 运行 `whoami` → 预期返回容器用户
3. 运行 `claude --version` → 预期输出版本号
4. 运行 `echo hello` → 预期正常输出

**预期：** 终端交互正常，claude 命令可用。

**步骤 4: 端口转发（Happy Path 步骤 4）**

1. 在 VS Code 终端中运行：`python3 -m http.server 8888 &`
2. 观察 VS Code 是否自动检测端口转发
3. 在宿主机浏览器访问 `http://localhost:8888`

**预期：** VS Code 提示端口已转发，浏览器可访问。

**步骤 5: 扩展安装（Happy Path 步骤 5）**

1. VS Code → 扩展面板
2. 搜索 "Go" 或 "Python"
3. 在远程侧安装扩展

**预期：** 扩展下载并安装成功。

**步骤 6-9: Egress 强约束验证**

按 `40-MANUAL-CHECKLIST.md` 中的 Egress 测试步骤执行：
6. 出口 IP 验证（终端 curl ifconfig.me）
7. DNS 泄漏验证（终端 nslookup）
8. VS Code 更新流量验证（sing-box 日志）
9. 端口转发出口 IP 验证
