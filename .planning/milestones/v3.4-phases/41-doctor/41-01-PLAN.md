### Task 4: remote-ssh 维度实现
**New file:** `internal/cloudclaude/doctor/remote_ssh.go`

5 个 check 函数，统一使用 `runWithTimeout` + `RemoteRunner`：

1. `checkVSCodeServerProcess(ctx, runner)` → `vscode_server_process`
   - `pgrep -f vscode-server`
   - 无进程 → Skip；有进程 → Pass

2. `checkVSCodeServerPort(ctx, runner)` → `vscode_server_port`
   - `ss -tlnp 2>/dev/null | grep -q vscode-server`
   - 端口未监听 → Warn(SSH_VSCODE_PORT_NOT_LISTENING)；正常 → Pass

3. `checkVSCodeServerDisk(ctx, runner)` → `vscode_server_disk`
   - `du -sh ~/.vscode-server/ 2>/dev/null`
   - 目录不存在 → Skip
   - 复用 `parseDuHumanToMB()` 解析
   - ≥2048MB → Fail(DISK_VSCODE_SERVER_BLOAT) + 清理建议
   - ≥500MB → Warn(DISK_VSCODE_SERVER_WARN) + 清理建议
   - <500MB → Pass

4. `checkForwardingSocket(ctx, runner)` → `forwarding_socket`
   - `ss -xp 2>/dev/null | grep -q forwarding`
   - socket 不存在 → Skip(SSH_FORWARDING_SOCKET_MISSING)
   - socket 存在 → Pass

5. `checkForwardingBlocked(ctx, runner)` → `forwarding_blocked`
   - `iptables -L OUTPUT -n 2>/dev/null | grep -c DROP`
   - 有 DROP 规则 → Warn(SSH_FORWARDING_BLOCKED)；无规则 → Pass

每个 check 函数开头做 `runner == nil` → Skip 降级。

        func(c context.Context) Check { return checkVSCodeServerProcess(c, remoteRunner) }))

        func(c context.Context) Check { return checkVSCodeServerDisk(c, remoteRunner) }))
