## Objective
为 remote-ssh 维度编写单元测试（fakeRunner mock），验证错误码注册和 explain 覆盖完整性。

## Tasks

### Task 1: remote-ssh 维度单元测试
**New file:** `internal/cloudclaude/doctor/remote_ssh_test.go`

使用 fakeRunner（实现 RemoteRunner 接口）mock 远端命令输出。

**Test cases:**

1. `TestCheckVSCodeServerProcess`
   - fakeRunner 返回 stdout 非空（进程存在）→ Pass
   - fakeRunner 返回 err（进程不存在）→ Skip

2. `TestCheckVSCodeServerPort`
   - fakeRunner 返回含 vscode-server 的 ss 输出 → Pass
   - fakeRunner 返回不含 vscode-server 的 ss 输出 → Warn(SSH_VSCODE_PORT_NOT_LISTENING)

3. `TestCheckVSCodeServerDisk`
   - fakeRunner 返回 `1.2G` → Fail(DISK_VSCODE_SERVER_BLOAT) + Details 包含清理建议
   - fakeRunner 返回 `800M` → Warn(DISK_VSCODE_SERVER_WARN) + Details 包含清理建议
   - fakeRunner 返回 `200M` → Pass
   - fakeRunner 返回 err（目录不存在）→ Skip

4. `TestCheckForwardingSocket`
   - fakeRunner 返回含 forwarding 的 ss 输出 → Pass
   - fakeRunner 返回 err（socket 不存在）→ Skip(SSH_FORWARDING_SOCKET_MISSING)

5. `TestCheckForwardingBlocked`
   - fakeRunner 返回 `3`（有 DROP 规则）→ Warn(SSH_FORWARDING_BLOCKED)
   - fakeRunner 返回 `0` → Pass

6. `TestCheckVSCodeServerPort_RequiresProcess` (可选)
   - 验证端口检测依赖进程检测的前置条件
