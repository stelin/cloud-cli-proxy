**3. errcodes/explanations.go — 在 SSH_SSHD_KEEPALIVE_DRIFT 的 registerExplanation 之后追加 3 条（每条 ≥200 中文字符）：**

- `SSH_SSHD_FORWARDING_DISABLED`: 触发场景为 AllowTcpForwarding 缺失或 no → VS Code Remote SSH 端口转发（direct-tcpip / tcpip-forward）完全不可用。根本原因是 sshd_config 配置被手动修改或镜像版本不对。修复路径为检查 deploy/docker/managed-user/sshd_config 中 `AllowTcpForwarding yes`，重建容器恢复。
- `SSH_SSHD_STREAM_FORWARDING_DISABLED`: 触发场景为 AllowStreamLocalForwarding 缺失或 no → Unix domain socket 转发不可用。影响 VS Code 的 forwarding socket（如 /tmp/vscode-ssh-proxy.sock）。修复路径与上同。
- `SSH_SSHD_GATEWAY_PORTS_OPEN`: 触发场景为 GatewayPorts 非 no → 远程端口转发绑定到 0.0.0.0，外部可访问容器内转发端口，违反安全约束。根本原因可能是配置被误改。修复路径为将 GatewayPorts 改为 no 并重启 sshd。

**4. doctor/ssh.go — 新增两个函数（parseSSHDKeepalive 函数之后）：**

`parseSSHDForwarding` 函数签名：
```go
func parseSSHDForwarding(out string) (tcpForwarding, streamForwarding, gatewayPorts string)
```
实现：复用 `parseSSHDKeepalive` 的 strings.Split / HasPrefix / TrimPrefix 模式。对三个指令分别匹配 `allowtcpforwarding `、`allowstreamlocalforwarding `、`gatewayports `（全部小写，防御性处理：strings.ToLower(line)）。返回原始字符串值（yes/no/other），缺失时返回空串。

`checkSSHDForwarding` 函数签名：
```go
func checkSSHDForwarding(ctx context.Context, runner RemoteRunner) Check
```
实现：复用 `checkSSHDKeepaliveDrift` 模式：
- runner == nil → return newSkip("ssh", "sshd_forwarding", "未能连接远端容器，跳过")
- 调用 `runner.RunScript("sshd_forwarding", "sshd -T 2>/dev/null | grep -E '^(allowtcpforwarding|allowstreamlocalforwarding|gatewayports)\\b' || true")`
- 调用 `parseSSHDForwarding(stdout)` 解析
- 检查顺序（per D-01/D-02）：
  1. tcpForwarding != "yes" → return newWarn("ssh", "sshd_forwarding_disabled", errcodes.SSH_SSHD_FORWARDING_DISABLED, actualValue)
  2. streamForwarding != "yes" → return newWarn("ssh", "sshd_stream_forwarding_disabled", errcodes.SSH_SSHD_STREAM_FORWARDING_DISABLED, actualValue)
  3. gatewayPorts != "no" → return newWarn("ssh", "sshd_gateway_ports_open", errcodes.SSH_SSHD_GATEWAY_PORTS_OPEN, actualValue)
  4. 全部正确 → return newPass("ssh", "sshd_forwarding", "AllowTcpForwarding=yes AllowStreamLocalForwarding=yes GatewayPorts=no")
