# Milestone v3.5: 网络白名单与 DNS 拆分解析

**Status:** SHIPPED 2026-05-13
**Phases:** 45-47 (3 phases)
**Total Plans:** 10
**Tag:** v3.5

## Overview

让管理员能在后台对每个 host 配置代理白名单（IP / CIDR / 域名 / 端口），使特定流量绕过 sing-box tun 隧道走宿主机直连，同时保留 zero DNS leak 的安全承诺，并把「10 条安全不变量」固化为 CI 守护。

里程碑通过三段式落地：

- **主线 A** · sing-box 两段式静态配置改造 + 拆分 DNS 模型 + 容器 `/etc/resolv.conf` 接管 + 五张白名单表 + `loopback`/`lan` 内置预设。
- **主线 B** · 管理员 API（CRUD + preview + apply + rollback + effective + audit-log）+ 5 硬 1 软护栏 + React 后台 Bypass Tab（含 PreviewSheet / ApplyProgressDialog / RollbackConfirmDialog）。
- **主线 C** · Agent 热更新链路（rule-set 原子写 + nft -f 事务 + sing-box 文件 watch reload + 健康检查 3 次失败自动回滚）+ fail-closed 加固（uid 锁 + mDNS/LLMNR/NetBIOS drop + IPv6 双保险）+ E2E 流量验证（10 条安全不变量 CI 化）。

## Key Accomplishments

1. **sing-box 两段式渲染 + 拆分 DNS：** gateway 静态 `config.json` 改造为 `rule_set`（type=local，format=source）引用 + 7 条 route.rules（sniff → hijack-dns → ip_is_private → rule_set → final proxy-out）+ `tun` 启用 `strict_route`/`auto_route`/`endpoint_independent_nat`；`dns.servers` 同时提供 `dns-local`（type=local）与 `dns-proxy`（type=https，server=1.1.1.1，detour=proxy-out，domain_resolver=dns-local）；`.lan`/`.local`/`.internal` 后缀走 dns-local，公网白名单域名走 DoH，保护查询隐私。
2. **容器 DNS 入口锁 + Provider 拆分：** worker 容器 `/etc/resolv.conf` 与 `/etc/nsswitch.conf` 改为 `:ro` bind mount，唯一 nameserver 为 sing-box tun IP `172.19.0.1`（附 `ndots:0 single-request-reopen`），hosts 行强制 `files dns`；`ContainerProxyProvider` 拆分为 `PrepareGateway`（worker 创建前起 gateway + 写 DNS 源文件）+ `PrepareHost`（worker 创建后 connect netns + configure routes），消除 entrypoint 启动时 tun0 未监听的竞争窗口。
3. **白名单数据模型 + 系统预设：** migration `0019_host_bypass_rules.sql` 落地五张表（host_bypass_presets / rules / bindings / snapshots / audit_log），UUID 主键 + JSONB CHECK + TIMESTAMPTZ NOW；migration `0020_host_bypass_snapshot_source.sql` 加 source 列；Repository 19 个 Bypass* 方法 + 21 SQL 常量 + `ErrSystemBypassPresetImmutable` sentinel + Go/SQL 双层防御；seed 注入 `loopback`（is_system=true、is_force_on=true，不可关闭）和 `lan`（is_system=true，默认关闭）。
4. **18 条 Admin Bypass API + 6 护栏 + 双轨审计：** preset / rule / binding / validate / preview / apply / rollback / effective / audit-log / consistency 18 条路由完整接线；5 条硬护栏（`BYPASS_RULE_TOO_BROAD` / `BYPASS_RULE_CONFLICT_PROXY` / `BYPASS_LIMIT_EXCEEDED` 等 12 个错误码）+ `domain_keyword < 4` 软护栏（要求 `confirm_risky:true`）；所有写操作走 `writeBypassAuditLog` 双轨（DB audit_log 行 + EventRecorder 事件流）。
5. **Admin React Bypass Tab + 预览/应用/回滚三件套：** host 详情页新增「代理白名单」Tab，含预设网格（loopback 强制锁定 / lan 可勾选）+ 自定义规则表（五种类型 CRUD + 高风险徽章 + 二次确认）+ PreviewSheet（sing-box JSON 与 nft set diff 双 Tab 视图）+ ApplyProgressDialog（5 阶段固定中文标签 + task.progress_percent 4 档映射）+ RollbackConfirmDialog（host slug 严格输入），11 个 `.tsx` 组件 + 6 个 vitest 测试 + 4 个 hooks + 8 条中文错误码 i18n，34 个 case 全部 PASS。
6. **Agent 热更新链路 + fail-closed 流量加固 + 10 条不变量 CI：** worker dispatcher 真实接管 `ActionReloadHostBypass`，host-agent 完成「nft -f 事务更新 `@whitelist_v4` set → tmpfile+rename 原子写 rule-set 文件 → 等 1s sing-box 文件 watch reload → 健康检查 3 次失败自动 rollback 上一 applied snapshot」全流程，长跑 SSH 不断；`Consistency` 端点对账 nft set 与 rule-set 文件 SHA-256 归一化 hash；netns nft 四层防御（`oifname sb-tun0` accept + uid=singbox 直连代理 IP:443 + `oifname eth0 ip daddr @whitelist_v4` 逃逸通道 + 链末 `counter log prefix "sbfw-drop " drop`）+ 显式 drop mDNS/LLMNR/NetBIOS + 容器 `--sysctl disable_ipv6=1` 双保险；`verify.go` 新增 3 项流量检查（白名单走 eth0 / 非白名单走代理 / `dig @8.8.8.8` 必超时）；`scripts/uat-bypass.sh` 6 场景 × 10 不变量（I1–I10）+ `.github/workflows/uat-bypass.yml` fixture 自适应 preflight + `scripts/uat-bypass-fixture-up.sh` 三件套，CI 守护从「lint 就位」升级为「fixture 一旦合入即真实跑通」。

### Phase 45: 网络配置基础与数据模型

**Goal**: 让 sing-box 静态配置改造为两段式 + 拆分 DNS 模型 + 容器 resolv.conf 接管 + 白名单数据模型与系统预设全部就绪，为后续 API 和热更新打下可渲染、可入库、可挂载的基础。

**Depends on**: Phase 44 (v3.4 多形态容器接入 已交付的 sing-box gateway 基线)

**Requirements**: BYPASS-NET-01..04 / BYPASS-DNS-01..04 / BYPASS-DATA-01..04

**Plans**: 3 plans

- [x] 45-01-PLAN.md — sing-box 配置渲染层扩展（两段式 + 拆分 DNS + tun 加固）+ rule-set placeholder 写盘与挂载 — completed 2026-05-12
- [x] 45-02-PLAN.md — 容器 DNS 入口锁（resolv.conf / nsswitch.conf ro bind mount + verify 期望常量化）+ Provider 拆分（PrepareGateway + PrepareHost）— completed 2026-05-12
- [x] 45-03-PLAN.md — migration 0019 五张表 + 两条系统预设 seed + Repository 19 个 CRUD 方法 + ErrSystemBypassPresetImmutable 双层防御 — completed 2026-05-12

**Details:**

1. 新建 host 渲染出的 `config.json` 包含 `route.rule_set` 数组（`whitelist-cidrs` / `whitelist-domains`，type=local，format=source），`route.rules` 第一条为 `sniff`（sniffer 至少含 `tls/http/quic/dns`），含 `protocol:"dns"` → `hijack-dns` 与 `ip_is_private:true` → `direct-out` 兜底；`route.final = "proxy-out"`、`route.default_interface = "eth0"`，`tun` inbound 同时启用 `strict_route` / `auto_route` / `endpoint_independent_nat`。
2. sing-box `dns` 段同时存在 `dns-local`（type=local）和 `dns-proxy`（type=https，server=1.1.1.1，detour=proxy-out，domain_resolver=dns-local），`dns.final = "dns-proxy"`、`strategy = "ipv4_only"`；`.lan` / `.local` / `.internal` 后缀走 `dns-local`，命中 `whitelist-domains` rule_set 的域名走 `dns-proxy`。
3. worker 容器启动后，`nsenter -t <PID> -m cat /etc/resolv.conf` 唯一 nameserver 为 `172.19.0.1`（含 `options ndots:0 single-request-reopen`），并且文件以只读 bind mount 挂载；`/etc/nsswitch.conf` 的 `hosts:` 行为 `files dns`，无 mdns/myhostname/wins。
4. 数据库迁移 `0019_host_bypass_rules.sql` 应用后，五张表 + `loopback`（is_system=true、is_force_on=true）和 `lan`（is_system=true、is_force_on=false）两条系统预设 seed 全部存在。
5. Repository 层 `BypassPreset` / `BypassRule` / `BypassBinding` / `BypassSnapshot` 的 19 个 CRUD 方法 + 21 SQL 常量 + `ErrSystemBypassPresetImmutable` sentinel 全部 PASS；`host_bypass_snapshots.applied_status` 默认值为 `pending`，可被写入 `applied` / `failed` / `rolled_back` 四态。

**Details:**

**Key Decisions:**

- 两段式 sing-box 配置：静态 `config.json` 模板渲染（每 host 一份，变更需重启）+ 动态 local rule-set 文件（sing-box 1.10+ 文件 watch 自动 reload，不重启进程）。
- 拆分 DNS：内网 `.lan/.local/.internal` 走 `dns-local`，公网白名单走代理 DoH，即使流量走 direct 也不让 LAN DNS 留痕。
- 容器 `/etc/resolv.conf` 改为只读挂载指向 sing-box tun IP（`172.19.0.1`），替代旧 8.8.8.8 占位。
- `ContainerProxyProvider.PrepareHost` 拆分为 `PrepareGateway` + `PrepareHost`，让 worker 容器创建前 gateway 已起、DNS 源文件已写。
- `GatewayConfigDir` 提升为导出包级函数，单点事实源，避免循环依赖。
- IPv6：v3.5 容器内全禁（`--sysctl disable_ipv6=1` + nft IPv6 family policy=drop 等价 ip6tables drop）。
- fail-closed：sing-box 起不来则容器 unhealthy 不开 SSH；nft `output policy drop` + `oifname sb-tun0` accept + uid=singbox 锁定直连代理 IP:443 + `@whitelist_v4` 逃逸通道 + 链末 `counter log drop`。
- v3.5 仅管理员可配置，不做用户自助；系统预设仅 `loopback`（强制开启）+ `lan`（默认关闭）；`cn-dev` / `oss-dev` / `ai-api` 推到 P1。
- 用 TEXT + CHECK 约束代替 PostgreSQL ENUM，对齐项目历史。
- `is_system` 双层防御：Go 层 sentinel `ErrSystemBypassPresetImmutable` + SQL `WHERE is_system = FALSE` 兜底，handler `errors.Is` 翻译成 HTTP 403。
- 双轨审计：写操作成功后串行调用 `InsertBypassAuditLog` → `EventRecorder.RecordEvent`，任一失败仅 Warn 不阻断主请求。
- `QueueHostAction` 第 5 参用专属 `bypassSnapshotID` 形参，类型签名层面把「借 requestedBy 传 snapshot ID」hack 关闭。
- `ApplyBypassRuleSet` 严格顺序「先 nft 事务 → 后 atomic write」：nft 失败不动文件，避免「文件已变但 nft 未变」漂移。
- 健康检查复用容器 namespace 内 `/dev/tcp/192.168.0.1/53` TCP 半握手 —— 既不发真实 DNS 报文，也能验证 sb-tun0 路由是否生效。
- rollback 不修改 target snapshot 的 applied_status：新建 `source='rollback'` pending 行，由 worker 决定状态切换；hash 后缀用 `crypto/rand` 8 字节 hex 绕开 UNIQUE 冲突且不污染 nft 标识。
- rollback 路径不走 admin API 重入，直接在 worker 内复用 `ApplyBypassRuleSet` 重下发上一 applied snapshot。
- `Consistency` endpoint 包 3s `context.WithTimeout` 防 nft 卡死 DoS（504 BYPASS_CONSISTENCY_TIMEOUT vs. 500 BYPASS_CONSISTENCY_ERROR 分开）。
- nft 表族 = `ip cloudproxy`（与 47-02 现状对齐），`nftRunner` / `nftJSONLister` 自带 `nsenter -t <pid> -n --` 包装，单点事实源在 `bypassNftFamily` / `bypassNftTable` 常量。
- 用 plan-descriptor 层做规则顺序单测，避免 mock `*nftables.Conn`。
- `nsenterRunner` 抽包级 var（与 `bypass_reload.go::nftRunner` 同模式），让单测旁路 exec 完全 in-process，CI 不需要 root/Linux 也能跑 verify_test 单元层。
- uat-bypass.yml uat job 不用 `if: false`，改为 fixture 自适应 preflight：fixture 缺失时 job 跳过并发 warning，fixture 一旦合入 PR 即自动触发 6 场景 × ubuntu-24.04 × I1–I10 CI 守护。
