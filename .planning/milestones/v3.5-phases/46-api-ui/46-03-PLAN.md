## API 路由（Plan 01）

```
GET    /v1/admin/bypass/presets                                → { presets: BypassPreset[] }
GET    /v1/admin/bypass/presets/{presetID}                     → { preset: BypassPreset }
POST   /v1/admin/bypass/presets                                → { preset: BypassPreset }
PATCH  /v1/admin/bypass/presets/{presetID}                     → { preset: BypassPreset } | 403 BYPASS_PRESET_IMMUTABLE
DELETE /v1/admin/bypass/presets/{presetID}                     → 204                       | 403 BYPASS_PRESET_IMMUTABLE

GET    /v1/admin/bypass/rules?host_id=<id>                     → { rules: BypassRule[] }
POST   /v1/admin/bypass/rules                                  → { rule: BypassRule }      | 422 BYPASS_RULE_* / 400 BYPASS_KEYWORD_TOO_SHORT
POST   /v1/admin/bypass/rules/validate                         → { valid, is_risky, code?, message? }
PATCH  /v1/admin/bypass/rules/{ruleID}                         → { rule: BypassRule }
DELETE /v1/admin/bypass/rules/{ruleID}                         → 204

GET    /v1/admin/hosts/{hostID}/bypass                         → { bindings: BypassBinding[] }
POST   /v1/admin/hosts/{hostID}/bypass                         → { binding: BypassBinding } body: { preset_id?, rule_id?, enabled, source? }
DELETE /v1/admin/bypass/bindings/{bindingID}                   → 204
```

## 类型对齐（Plan 01 已确认 snake_case 字段名）

```ts
interface BypassPresetRule { rule_type: string; value: string; note?: string }
interface BypassPreset    { id, slug, name, description: string; is_system, is_force_on, is_active: boolean; rules: BypassPresetRule[]; created_at, updated_at: string }
interface BypassRule      { id, scope: "global"|"host"; host_id: string|null; rule_type, value, note: string; is_risky: boolean; created_at, updated_at: string }
interface BypassBinding   { id, host_id: string; preset_id: string|null; rule_id: string|null; enabled: boolean; source: string; created_at: string }
```

## 错误响应格式

后端任何 4xx/5xx 都返回 `{ code: "BYPASS_*", message: "..." }`（Plan 01 写入 writeBypassError）。前端 `apiFetch` 现状在 `!res.ok` 时 `throw new ApiError(res.status, body)`（body 是 raw text）—— 本 plan 不修改 apiFetch，改在 hook 层用 try/catch 解析 body json 取 code。
