# Plan 54-01: container_proxy_provider 单容器路径重构

## Objective

把 `internal/network/container_proxy_provider.go` 从"sidecar gateway 容器 + 自定义 bridge + worker netns 注入"模型重构为"单容器、host-agent 仅写 sing-box config + verify"的最小路径，对应 v4.0 ROADMAP `Phase 54 Success Criteria` 第 1 / 2 / 5 条的代码层落地。

完成后：
- `PrepareGateway` 不再创建 docker 网络、不再 `docker run` gateway 容器、不再写 placeholder ruleset，**仅作为"sing-box config 注入入口的占位"**（实际写文件逻辑在 Plan 54-02）。
- `PrepareHost` 不再 `dockerNetworkConnect` / `disconnect bridge` / `configureWorkerEgress` / `applyWorkerFirewall` / 控制面 join 隔离网络；只保留 `verifyWorkerNetwork`（容器内 sing-box 自起 tun，无需 host 端配路由）。
- `teardownGateway` 整段删除；`CleanupHost` 简化为"删 host 端 sing-box config 目录"。
- `worker.go::buildCreateArgs` 删除 v3.5 双源 DNS bind mount，新增 `--device /dev/net/tun` + `--restart=on-failure` + sing-box config bind mount 占位（实际 bind mount path 由 54-02 决定）。

```go
// PrepareGateway 在 docker create 之前把 sing-box config 写到 host 端
// SingBoxConfigDir(hostID)/config.json，权限 root:singbox 0640（D-54-2 / Plan 54-02）。
// 容器随后通过 ro bind mount 把该文件挂到 /etc/sing-box/config.json，
// entrypoint start_singbox_or_die 读取后 shred -u 从 fs 删除（D-V4-2）。
//
// v4.0 (Phase 54) 改造：不再创建 cloudproxy-net-* 自定义 bridge，
// 不再起 sidecar gateway 容器；user 容器自带 sing-box（Phase 53 镜像）。
func (p *ContainerProxyProvider) PrepareGateway(ctx context.Context, spec HostNetworkSpec) error {
    if spec.Egress == nil {
        p.logger.Info("container-proxy: no egress config, skipping config inject", "host_id", spec.HostID)
        return nil
    }
    if spec.Egress.Proxy == nil {
        p.logger.Warn("container-proxy: no proxy config, skipping config inject", "host_id", spec.HostID)
        return nil
    }

    dir := SingBoxConfigDir(spec.HostID)
    if err := os.MkdirAll(dir, 0o755); err != nil {
        return fmt.Errorf("sing-box: mkdir config dir: %w", err)
    }
    // Plan 54-02 实现：构建 + 写 config（root:singbox 0640）
    if err := writeContainerSingBoxConfig(spec.HostID, spec.Egress); err != nil {
        return fmt.Errorf("sing-box: write config: %w", err)
    }
    p.logger.Info("container-proxy: sing-box config injected", "host_id", spec.HostID, "dir", dir)
    return nil
}
```

### T2 — 改造 `PrepareHost` 删 host 端网络/防火墙路径

文件：`internal/network/container_proxy_provider.go`

新 `PrepareHost` 骨架（与现有 ~80 行对比，精简到 ~30 行）：

```go
// PrepareHost 在 worker 容器 docker start 之后调用。
// v4.0 (Phase 54): user 容器自带 sing-box（Phase 53），host-agent 不再
// 进入 worker netns 跑 ApplyWorkerFirewallRules（entrypoint apply_nft_or_die
// 在容器内部应用）；不再 dockerNetworkConnect / disconnect bridge / configure
// route（容器走默认 docker bridge + 自带 tun0 路由）；不再把控制面 join 隔离
// 网络（无隔离网络存在）。仅保留 verifyWorkerNetwork 做出口 IP / DNS / leak
// 三检，确认 Phase 53 entrypoint 启动序列真的把流量导到 tun0。
func (p *ContainerProxyProvider) PrepareHost(ctx context.Context, spec HostNetworkSpec) error {
    if spec.Egress == nil {
        p.logger.Info("container-proxy: no egress config, skipping verify", "host_id", spec.HostID)
        return nil
    }
    if spec.Egress.Proxy == nil {
        p.logger.Warn("container-proxy: no proxy config, skipping verify", "host_id", spec.HostID)
        return nil
    }

    workerName := workerContainerName(spec.HostID)
    result, verifyErr := verifyWorkerNetwork(ctx, workerName, *spec.Egress)
    if verifyErr != nil {
        p.logger.Error("container-proxy: network verification failed",
            "host_id", spec.HostID,
            "egress_ip_match", result.EgressIPMatch,
            "dns_correct", result.DNSCorrect,
            "leak_blocked", result.LeakBlocked,
            "actual_egress_ip", result.ActualEgressIP,
            "actual_dns", result.ActualDNS,
        )
        if netErr, ok := verifyErr.(*NetworkError); ok {
            netErr.HostID = spec.HostID
        }
        return verifyErr
    }
    p.logger.Info("container-proxy: network verification passed (single-container)",
        "host_id", spec.HostID,
        "egress_ip", result.ActualEgressIP,
        "dns_server", result.ActualDNS,
    )
    return nil
}
```

删除：v3.5 `dockerNetworkConnect` worker → 自定义 net、`disconnect bridge`、`configureWorkerEgress`、`applyWorkerFirewall`、控制面 join 隔离 net 全段。

```go
// SingBoxConfigDir 返回 host 专属的 sing-box config 目录。
// 路径规则：<DATA_DIR>/gateway/<host_id>/（路径名"gateway"是 v3.x 历史
```
