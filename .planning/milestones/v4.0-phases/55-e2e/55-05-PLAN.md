# Plan 55-05: 安全断言 SEC-01..03

## Objective

新增三条同容器架构特有的安全断言 e2e 用例（对应 IMG-04 的验证），确保用户进程无法破坏网络隔离：

- **SEC-01**: 用户不能 kill sing-box
- **SEC-02**: 用户不能读 sing-box config
- **SEC-03**: 用户 cap 集合为空

## Scope

| REQ | 实现位置 | 验收 |
|---|---|---|
| SEC-01 | `tests/e2e/security_cannot_kill_singbox_test.go` | `docker exec -u user kill -9 $(pidof sing-box)` → `Operation not permitted` |
| SEC-02 | `tests/e2e/security_cannot_read_config_test.go` | `docker exec -u user cat /etc/sing-box/config.json` → `Permission denied` 或 `No such file` |
| SEC-03 | `tests/e2e/security_empty_cap_test.go` | `docker exec -u user getpcaps $$` → 空 cap 集合 |

## Files Touched

- `tests/e2e/security_cannot_read_config_test.go` — **新建**
