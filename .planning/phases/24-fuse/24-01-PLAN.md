  <action>
    **Dockerfile 修改（per D-01, D-02）：**

    1. 在第 9-39 行的 apt-get install 块中，在 `xterm \` 之后、`&& rm -rf` 之前追加两个包：
       ```
       sshfs \
       fuse3 \
       ```

    2. 在该 apt-get install 的 `RUN` 指令之后（第 39 行 `&& rm -rf /var/lib/apt/lists/*` 之后），新增一个独立 RUN 指令配置 fuse.conf：
       ```dockerfile
       RUN sed -i 's/^#user_allow_other/user_allow_other/' /etc/fuse.conf \
           && chmod a+r /etc/fuse.conf
       ```
       - `sed` 取消注释 user_allow_other，允许非 root 用户使用 `-o allow_other` 挂载
       - `chmod a+r` 确保非 root 用户可以读取 fuse.conf（某些发行版默认不可读）

    **entrypoint.sh 修改：**

    3. 在第 93 行（IPv6 禁用 `sysctl` 之后）和第 95 行（KasmVNC 配置 `# KasmVNC 配置` 注释之前）之间，插入 /dev/fuse 权限修复代码块：
       ```bash
       if [ -c /dev/fuse ]; then
         chmod 666 /dev/fuse
       fi
       ```
       - 使用 `[ -c /dev/fuse ]` 条件判断，兼容没有 `--device /dev/fuse` 的旧容器
       - Docker `--device` 注入的 /dev/fuse 默认权限为 `crw-rw---- root:root`，workspace 用户无法访问
       - entrypoint.sh 以 root 身份运行，是修复此权限的正确位置
  </action>
