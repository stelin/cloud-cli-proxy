<objective>
在受管镜像 `entrypoint.sh` 中，紧随现有 `chpasswd` 调用之后增加一段 `passwd -S` 自检：当目标用户密码状态不是 `P` / `PS` 时，立即 `exit 1`。这是 Phase 29.1 的镜像侧防御——即使上游（Plan 01 仓储层 / Plan 02 worker）在未来发生回归，容器宁可起不来，也不起一个登不进去的"伪成功"容器。

Purpose: 把"密码退化为 workspace"类故障从"用户排障"提前到"容器启动失败"；结合 docker restart 策略能让此类故障立即可见。
Output: 一处 entrypoint.sh 微改；对其他段零侵扰；镜像 build & 容器启动路径手动验证通过。
</objective>

      - 其余值（包括 `L` / `LK` / `NP` / `UNSET` / 空）→ stderr 输出 FATAL 消息并 `exit 1`；

    在 `deploy/docker/managed-user/entrypoint.sh` 的 `echo "${RUN_USER}:${CONTAINER_PASSWORD}" | chpasswd` 这一行**紧随之后**、当前 IPv6 禁用段 `sysctl -w net.ipv6.conf.all.disable_ipv6=1 ...` **之前**，插入如下代码块：

    ```bash
    # Phase 29.1: 验证密码已被成功设置，避免"密码退化为 workspace"的静默失败复现。
    # passwd -S 第 2 列语义：P / PS = 已设置密码；L / LK = 已锁定；NP = 无密码；UNSET = 读取失败。
    status="$(passwd -S "${RUN_USER}" 2>/dev/null | awk '{print $2}' || echo UNSET)"
    case "$status" in
      P|PS)
        ;;
      *)
        echo "[entrypoint] FATAL: password status for ${RUN_USER} is '${status}', refusing to start" >&2
        exit 1
        ;;
    esac
    ```

    严禁：
    - 修改此段之外的任何代码行（包括 chpasswd 行本身、IPv6 禁用、sudoers、mkdir / chown、write_desktop_config、KasmVNC、prepare_v3_* 等）；
    - 在 FATAL 消息中回显 `CONTAINER_PASSWORD` 或 `CONTAINER_SSH_PASSWORD` 值；
    - 删除或调整上方 `CONTAINER_PASSWORD="${CONTAINER_SSH_PASSWORD:-workspace}"` 这一行（入口脚本要保证变量总能展开，fail-fast 由 passwd -S 负责）；
    - 把 `awk` / `passwd` 替换成别的工具（镜像内 util-linux 已包含，无需新依赖）。

    如果发现脚本顶部缺少 `set -u`，**保持现状**；本任务不做脚本风格重构。
