# Phase 52 Plan 01: `tests/e2e/harness/collect-artifacts.sh` 一键采集脚本 (OBS-01)

## Goal

落地 `tests/e2e/harness/collect-artifacts.sh`，**一条命令**把单次 e2e 用例失败时的全部排障证据采集到 `<output-dir>/<scenario-id>/{logs,network,docker,postgres,system}/`：

- 接口：`bash tests/e2e/harness/collect-artifacts.sh <output-dir> [scenario-id]`
- 错误容错：**任一子命令失败永远不让脚本返回非 0**；缺工具时跳过写空文件占位。
- darwin + ubuntu-24.04 双跑：darwin 因为缺 `docker logs / nft / pg_dump`，对应文件会是空 / 错误占位；ubuntu-24.04 CI runner 上是真实输出。
- 隐私守护：脚本输出绝不含 `/Users/`、`/home/<user>/`、个人邮箱等敏感字符串；所有路径用 `$OUT` 形参化。
- 配套 Go 单测 `tests/e2e/harness/collect-artifacts_test.go`（**无 build tag**，darwin 也跑）：`os/exec` 起脚本子进程 → 检查 5 子目录 + metadata.txt 存在。

**不在本 plan 范围**（属 Plan 02 / Plan 03）：

- 5 个 README 文案与 metadata.txt 内容细节（Plan 02 落地）。
- DumpHook 内部从「Go 实现」切到「子进程脚本」（Plan 03 落地）。
- `.github/workflows/e2e.yml` 改造（Plan 03 落地）。

## Steps

1. **新建 `tests/e2e/harness/collect-artifacts.sh`**

   骨架：

   ```bash
   #!/usr/bin/env bash
   # Phase 52 OBS-01：e2e 失败一键采集脚本
   #
   # 用法：
   #   bash tests/e2e/harness/collect-artifacts.sh <output-dir> [scenario-id]
   #
   # 行为：
   #   - 在 <output-dir>/<scenario-id>/ 下建 5 个子目录（logs/network/docker/postgres/system）
   #   - 每个子目录尝试运行对应采集命令，缺工具或失败时写空文件占位
   #   - 脚本永远 exit 0（除非参数缺失）
   #   - 设 COLLECT_DEBUG=1 打开 bash -x 调试

   set -uo pipefail   # 注意：不带 -e，允许子命令失败

   if [[ "${COLLECT_DEBUG:-0}" = "1" ]]; then set -x; fi

   OUT="${1:?需要 output-dir，用法：bash collect-artifacts.sh <output-dir> [scenario-id]}"
   SCENARIO="${2:-default}"
   ROOT="$OUT/$SCENARIO"

   mkdir -p "$ROOT/logs" "$ROOT/network" "$ROOT/docker" "$ROOT/postgres" "$ROOT/system"

   collect_metadata
   collect_logs
   collect_network
   collect_docker
   collect_postgres
   collect_system

   echo "[collect-artifacts] done: $ROOT"
   exit 0
   ```

   `collect_metadata`：写 `metadata.txt` 到 `$ROOT/`（**仅** Plan 01 留挂点，Plan 02 填实质内容）。本 plan 至少写 4 行：

   ```
   timestamp=<date -u +%Y-%m-%dT%H:%M:%SZ>
   scenario=$SCENARIO
   hostname=<hostname>
   kernel=<uname -srm>
   ```

   `collect_logs`（容器日志）：

   ```bash
   collect_logs() {
     if ! command -v docker >/dev/null 2>&1; then
       echo "docker not available" > "$ROOT/logs/_skipped.txt"
       return
     fi
     docker ps -a --format '{{.Names}}' 2>/dev/null | while read -r name; do
       [[ -z "$name" ]] && continue
       docker logs --tail 500 --timestamps "$name" \
         > "$ROOT/logs/${name}.log" 2>&1 || true
     done
   }
   ```

   `collect_network`（host root 网络状态）：

   ```bash
   collect_network() {
     # nft list ruleset
     if command -v nft >/dev/null 2>&1; then
       nft list ruleset > "$ROOT/network/nft-ruleset.txt" 2>&1 || true
     else
       echo "nft not available" > "$ROOT/network/nft-ruleset.txt"
     fi
     # ip 系列
     if command -v ip >/dev/null 2>&1; then
       ip -o link    > "$ROOT/network/ip-link.txt"  2>&1 || true
       ip -o addr    > "$ROOT/network/ip-addr.txt"  2>&1 || true
       ip route      > "$ROOT/network/ip-route.txt" 2>&1 || true
       ip netns list > "$ROOT/network/ip-netns.txt" 2>&1 || true
     else
       echo "ip not available" > "$ROOT/network/ip-link.txt"
     fi
     # netstat（备选 ss）
     if command -v ss >/dev/null 2>&1; then
       ss -tln > "$ROOT/network/listen-tcp.txt" 2>&1 || true
     elif command -v netstat >/dev/null 2>&1; then
       netstat -tln > "$ROOT/network/listen-tcp.txt" 2>&1 || true
     else
       echo "ss/netstat not available" > "$ROOT/network/listen-tcp.txt"
     fi
   }
   ```

   `collect_docker`（docker 元信息）：

   ```bash
   collect_docker() {
     if ! command -v docker >/dev/null 2>&1; then
       echo "docker not available" > "$ROOT/docker/_skipped.txt"
       return
     fi
     docker ps -a > "$ROOT/docker/ps.txt" 2>&1 || true
     docker network ls > "$ROOT/docker/network-ls.txt" 2>&1 || true
     docker ps -a --format '{{.Names}}' 2>/dev/null | while read -r name; do
       [[ -z "$name" ]] && continue
       docker inspect "$name" > "$ROOT/docker/inspect-${name}.json" 2>&1 || true
     done
   }
   ```

   `collect_postgres`（Postgres schema-only + key 表 SELECT）：

   ```bash
   collect_postgres() {
     # PG_DUMP_URL：e2e harness 启控制面前会 export DATABASE_URL；如无则跳过
     local url="${PG_DUMP_URL:-${DATABASE_URL:-}}"
     if [[ -z "$url" ]]; then
       echo "PG_DUMP_URL / DATABASE_URL 未设置，跳过 postgres 采集" \
         > "$ROOT/postgres/_skipped.txt"
       return
     fi
     if ! command -v pg_dump >/dev/null 2>&1; then
       echo "pg_dump not available" > "$ROOT/postgres/_skipped.txt"
       return
     fi
     # 仅 --schema-only，不含敏感行数据
     pg_dump --schema-only --no-owner --no-privileges "$url" \
       > "$ROOT/postgres/schema.sql" 2>"$ROOT/postgres/schema.err" || true
     # 几个 key 表的安全 SELECT（不含密码 / token）
     if command -v psql >/dev/null 2>&1; then
       psql "$url" -At -F$'\t' \
         -c "select id, username, status from users limit 50" \
         > "$ROOT/postgres/users.tsv" 2>"$ROOT/postgres/users.err" || true
       psql "$url" -At -F$'\t' \
         -c "select id, host_id, egress_ip_id, created_at from host_egress_bindings limit 50" \
         > "$ROOT/postgres/host-egress-bindings.tsv" \
         2>"$ROOT/postgres/host-egress-bindings.err" || true
       psql "$url" -At -F$'\t' \
         -c "select id, type, created_at from events order by id desc limit 100" \
         > "$ROOT/postgres/events.tsv" 2>"$ROOT/postgres/events.err" || true
     fi
   }
   ```

   `collect_system`（宿主机系统状态）：

   ```bash
   collect_system() {
     uname -a > "$ROOT/system/uname.txt" 2>&1 || true
     # free / df 在 macOS 上不同；缺工具就空占位
     if command -v free >/dev/null 2>&1; then
       free -m > "$ROOT/system/free.txt" 2>&1 || true
     else
       vm_stat > "$ROOT/system/free.txt" 2>&1 || true   # darwin 兜底
     fi
     df -h > "$ROOT/system/df.txt" 2>&1 || true
     # dmesg 最近 100 行（root 权限或 --read 才行）
     if command -v dmesg >/dev/null 2>&1; then
       dmesg --time-format=iso 2>/dev/null | tail -n 100 \
         > "$ROOT/system/dmesg-tail.txt" 2>&1 || \
       dmesg 2>/dev/null | tail -n 100 > "$ROOT/system/dmesg-tail.txt" 2>&1 || true
     else
       echo "dmesg not available" > "$ROOT/system/dmesg-tail.txt"
     fi
   }
   ```

   **隐私守护**：

   - 脚本本身禁出现 `/Users/`、`/home/<具体用户名>/`、个人邮箱、真实 IP / 域名等字面量。
   - 所有输出目录走 `$OUT` / `$ROOT` 形参化。
   - `pg_dump` 走 `--schema-only`；用户表只 SELECT `id / username / status`，**不**导出 `password_hash` / `entry_password` 等敏感列。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| darwin 没装 `nft / ip / pg_dump`，脚本误退非 0 | `set -uo pipefail` 但**不带** `-e`；每个子命令显式 `|| true`；缺工具走 `_skipped.txt` 占位 |
| `pg_dump` 没设 `DATABASE_URL` 导致 connection refused 把脚本拖慢 | 先判 `[[ -z "$url" ]]` 跳过；有 url 但 connect 失败时 `pg_dump` 自己 2 秒内 fail，并通过 `|| true` 吞掉错 |
| 脚本写了绝对路径（`/Users/...` 之类）被 CONVENTIONS.md §Privacy 守护拦下 | 所有路径走 `$OUT` / `$ROOT` 形参化；新增单测 `TestCollectArtifacts_NoAbsoluteUserPathsInScript` 静态扫脚本源码守护 |
| `docker logs <name>` 输出含 token / 用户密码 | e2e fixture 用 `test@example.com` / `secret-placeholder-pw` 占位（Phase 47 已锁），脚本不再额外过滤；后续 phase 如引入真实凭据，文档要求 SUMMARY 显式记录 |
| `dmesg` 在非 root 环境拒绝读 | 双层兜底（先 `--time-format=iso`，再 plain `dmesg`），最终失败仍 `|| true` 吞错；CI runner 通常 root，没问题 |
| 单测在 darwin 上偶发依赖 `docker daemon` 启停状态导致结果不稳 | 脚本对 `command -v docker` 检测后再 `docker ps`；单测只断言 5 子目录 + metadata.txt 存在，**不**对子目录内容做强校验（避免 darwin 上 `docker` 在 / 不在导致用例摆动）|
| 后续 phase 增 / 删工具命令（e.g. `iptables` vs `nft`） | 本 plan 锁 nft 优先 + ss/netstat 兜底；新工具入场时只在脚本内追加 `command -v X` 分支，不改 5 子目录契约 |
