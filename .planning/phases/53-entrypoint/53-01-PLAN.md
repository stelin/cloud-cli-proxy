# Plan 53-01: Dockerfile 改造

## Objective

把 `deploy/docker/managed-user/Dockerfile` 改造为 v4.0 单容器架构的镜像基线：内置 sing-box binary + 创建 `singbox` 系统账号 + 给 binary 打文件 cap + 删除 workspace 用户的 sudo NOPASSWD。

## Scope

| REQ | 实现位置 | 验收 |
|---|---|---|
| IMG-01 | Dockerfile 新增 sing-box install 段 | `/usr/local/bin/sing-box version` 输出 `sing-box version 1.13.3` |
| IMG-02 | Dockerfile 加 `useradd singbox` 段 | `id singbox` 返回 `uid=9000(singbox) gid=9000(singbox)` |
| IMG-03 | Dockerfile apt 包列表加 `nftables` + `libcap2-bin` | `command -v nft` 和 `command -v setcap` 都不为空 |
| IMG-04 | Dockerfile 删除 sudoers RUN 段 + 改 useradd 不加入 sudo group | `sudo -n true` 失败；`groups workspace` 不含 `sudo` |
| D-V4-1 / D-53-2 | Dockerfile 新增 setcap RUN | `getcap /usr/local/bin/sing-box` = `cap_net_admin,cap_net_bind_service=eip` |
| D-53-4 | Dockerfile 删除 sudoers 行 + entrypoint 改造（Plan 53-02 配合） | 容器内任何 user 不能 sudo |

## Tasks

### T1 — sing-box install 段

在 Dockerfile 中 Claude Code 安装段之后、SSH setup 之前插入：

```dockerfile
# sing-box: tun 模式同容器透明代理（v4.0）
ARG SINGBOX_VERSION=1.13.3
RUN set -eux; \
    ARCH="$(dpkg --print-architecture)"; \
    curl -fsSL -o /tmp/sb.tgz "https://github.com/SagerNet/sing-box/releases/download/v${SINGBOX_VERSION}/sing-box-${SINGBOX_VERSION}-linux-${ARCH}.tar.gz"; \
    tar -xzf /tmp/sb.tgz -C /tmp; \
    install -m 0755 "/tmp/sing-box-${SINGBOX_VERSION}-linux-${ARCH}/sing-box" /usr/local/bin/sing-box; \
    rm -rf /tmp/sb.tgz "/tmp/sing-box-${SINGBOX_VERSION}-linux-${ARCH}"; \
    sing-box version
```

**Rationale**: 与 `deploy/docker/sing-box-gateway/Dockerfile` 完全一致；版本 ARG 共享便于 Phase 54 退役 gateway 镜像时统一管理。

### T2 — `singbox` 系统账号

在 Dockerfile 用户创建段（当前 L104-109）改造：

```dockerfile
# v4.0: 删除 workspace 的 sudo NOPASSWD（IMG-04 / D-53-4）
RUN userdel -f ubuntu 2>/dev/null || true \
    && groupdel ubuntu 2>/dev/null || true \
    && groupadd --force --gid "${WORKSPACE_GID}" "${WORKSPACE_USER}" \
    && useradd --uid "${WORKSPACE_UID}" --gid "${WORKSPACE_GID}" --home-dir /workspace --create-home --shell /bin/bash "${WORKSPACE_USER}"
    # 注意：原 echo "${WORKSPACE_USER} ALL=(ALL) NOPASSWD:ALL" >/etc/sudoers.d/workspace 已删除

# v4.0: singbox 系统账号（uid=9000，固定，避免与未来用户 uid 冲突）（IMG-02）
RUN groupadd --force --gid 9000 singbox \
    && useradd --uid 9000 --gid 9000 --home-dir /nonexistent --no-create-home --shell /usr/sbin/nologin singbox
```

### T3 — nftables + libcap2-bin

在 Dockerfile 第一个 apt install 段（当前 L18-49）追加：

```dockerfile
    && apt-get install -y --no-install-recommends \
        ...existing packages... \
        nftables \
        libcap2-bin \
```

**Rationale**:
- `nftables` 提供 `nft` binary，entrypoint 应用 default-deny ruleset 需要（Plan 53-02）
- `libcap2-bin` 提供 `setcap` / `getcap` / `getpcaps`，IMG-04 / SEC-03 验证需要

### T4 — sing-box binary 文件 cap

在 T1 之后新增 RUN：

```dockerfile
# v4.0: sing-box 通过文件 cap 拿 NET_ADMIN，不需 root（D-V4-1 / D-53-2）
RUN setcap 'cap_net_admin+eip cap_net_bind_service+eip' /usr/local/bin/sing-box \
    && getcap /usr/local/bin/sing-box | grep -q 'cap_net_admin' \
    || { echo 'setcap failed' >&2; exit 1; }
```

**Rationale**:
- `cap_net_admin` 必须，建 tun 设备
- `cap_net_bind_service` 备用，sing-box 监听 <1024 端口时需要（DNS 53 在 stub 模式监听 127.0.0.1:53）
- `+eip` = Effective + Inheritable + Permitted
- build 期间立即 verify cap 已落到 binary 上

### T5 — Dockerfile 文档注释更新


## Risks

| Risk | Mitigation |
|---|---|
| `setcap` 在 overlay/squashfs 文件系统可能丢失 file cap | T4 build 期 `getcap` verify，CI 上 V3 二次 verify |
| Ubuntu 24.04 默认 `sudo` 包可能在 image 里仍存在（基础镜像残留） | V5 行为接受两种语义（找不到 binary 或被拒绝），都满足"用户不能成为 root" |
| arm64 / amd64 双架构 sing-box release URL 命名 | T1 用 `$(dpkg --print-architecture)`，与 sing-box-gateway Dockerfile 同模式（已在 v3.6 验证） |
