<task type="auto">
  <name>Task 2: 重写 migrator.go — database/sql + embed.FS + SQLite 语法</name>
  <files>internal/store/migrator/migrator.go</files>
  <read_first>internal/store/migrator/migrator.go</read_first>
  <action>
重写 migrator.go，完整替换为 database/sql 实现：

1. **导入变更**（per D-12）：
   - 移除 `github.com/jackc/pgx/v5/pgxpool`
   - 添加 `database/sql`
   - 添加 `embed`
   - 保留 `context`, `fmt`, `sort`, `strings`

2. **函数签名变更**（per D-02）：
   - 原：`func RunMigrations(ctx context.Context, db *pgxpool.Pool, dir string) error`
   - 新：`func RunMigrations(ctx context.Context, db *sql.DB, migrationFS embed.FS) error`
   - 使用 embed.FS 参数而非文件系统路径，便于调用方通过 `//go:embed` 注入

3. **schema_migrations 表**（per D-04, D-06）：
   ```sql
   CREATE TABLE IF NOT EXISTS schema_migrations (
       filename TEXT PRIMARY KEY,
       applied_at TEXT NOT NULL DEFAULT (CURRENT_TIMESTAMP)
   )
   ```
   - TIMESTAMPTZ → TEXT（per D-04）
   - NOW() → CURRENT_TIMESTAMP（SQLite 内置函数）

4. **迁移文件发现**：
   - 使用 `migrationFS.ReadDir(".")` 读取嵌入的 FS
   - 过滤 `.sql` 后缀
   - 按文件名排序（与原有 filepath.Glob 行为一致）

5. **查询改为 database/sql**（per D-11）：
   - `db.Exec(ctx, sql)` → `db.ExecContext(ctx, sql)`
   - `db.QueryRow(ctx, sql, args...).Scan(...)` → `db.QueryRowContext(ctx, sql, args...).Scan(...)`
   - `db.Begin(ctx)` → `db.BeginTx(ctx, nil)`
   - `$1` 占位符改为 `?`（SQLite 语法）

6. **SQL 常量**：全部 $N 改为 ?
   - schema_migrations CREATE: 无参数
   - SELECT EXISTS: `$1` → `?`
   - INSERT INTO schema_migrations: `$1` → `?`
  </action>
