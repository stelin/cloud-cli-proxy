对所有 21 个迁移文件执行以下系统性改写（per D-01, D-03~D-07）：

**类型映射规则（全局）：**
- `UUID PRIMARY KEY DEFAULT gen_random_uuid()` → `TEXT PRIMARY KEY`（per D-03，Go 侧 uuid.NewString() 生成，不在 SQL 层生成）
- `UUID`（非主键列，含 REFERENCES）→ `TEXT`
- `TIMESTAMPTZ NOT NULL DEFAULT NOW()` → `TEXT NOT NULL DEFAULT (CURRENT_TIMESTAMP)`（per D-04）
- `TIMESTAMPTZ` → `TEXT`
- `JSONB NOT NULL DEFAULT '...'::jsonb` → `TEXT NOT NULL DEFAULT '...'`（per D-05，移除 ::jsonb 类型转换）
- `JSONB` → `TEXT`
- `BOOLEAN NOT NULL DEFAULT FALSE` → `INTEGER NOT NULL DEFAULT 0`（per D-06）
- `BOOLEAN NOT NULL DEFAULT TRUE` → `INTEGER NOT NULL DEFAULT 1`
- `BOOLEAN` → `INTEGER`
- `SERIAL` / `BIGSERIAL` → `INTEGER PRIMARY KEY AUTOINCREMENT`（per D-07）
- `BIGINT` → `INTEGER`
- `INET` → `TEXT`（IP 地址以字符串存储）
- `INET NOT NULL UNIQUE` → `TEXT NOT NULL UNIQUE`

**PG 特有语法移除：**
- `CREATE EXTENSION IF NOT EXISTS pgcrypto;` — 删除整行（SQLite 不需要）
- `'...'::jsonb` — 移除 `::jsonb` 类型转换，保留字符串值
- `'...'::json` — 移除 `::json` 类型转换
- `NOW()` → `CURRENT_TIMESTAMP`（SQLite 内置）
- `ON CONFLICT (slug) DO NOTHING` — 保留（SQLite 支持）
- `CHECK (status IN (...))` — 保留（SQLite 支持）
- `DEFAULT gen_random_uuid()` — 删除（Go 侧生成 UUID）

**BEGIN/COMMIT 事务块：**
- 移除所有 `BEGIN;` 和 `COMMIT;`（migrator 自行管理事务，SQLite 不支持嵌套 BEGIN）
- 保留内部的 DDL 语句

**迁移文件特殊处理：**
- `0001_initial.sql`: 删除 `CREATE EXTENSION IF NOT EXISTS pgcrypto;`，所有 UUID → TEXT，TIMESTAMPTZ → TEXT，JSONB → TEXT，`NOW()` → `CURRENT_TIMESTAMP`
- `0005_detected_egress_ip.sql` + `0005_host_env_and_user_entry.sql`: 注意有两个编号 0005 的文件 — 确认 migrator 的排序逻辑能正确处理（按文件名字符串排序）
- `0012_ssh_keys_table.sql`: UUID → TEXT
- `0013_drop_wireguard.sql`: ALTER TABLE DROP COLUMN — SQLite 3.35+ 支持
- `0019_host_bypass_rules.sql`: 最复杂 — 5 张表 + 2 条 seed + JSONB → TEXT + BOOLEAN → INTEGER + 移除 BEGIN/COMMIT
- `0020_host_bypass_snapshot_source.sql`: ALTER TABLE ADD COLUMN — 检查是否需要 SQLite 兼容写法
- `0021_remove_ipv6_from_lan_preset.sql`: UPDATE JSONB 值 — 改为 TEXT 字符串操作
- `0022_host_resource_limits_nullable.sql`: ALTER COLUMN — SQLite 不支持 ALTER COLUMN，需用重建表模式或检查原始迁移是否可简化
- `0023_drop_bypass_created_by_fks.sql`: 涉及外键操作

**SQLite 不支持 ALTER TABLE DROP COLUMN（< 3.35）的处理：**
- 检查 0013 和 0023 中的 DROP COLUMN / 外键操作
- 如果目标 SQLite 版本 >= 3.35（modernc.org/sqlite 通常打包最新），可保留 DROP COLUMN
- 如果需要兼容旧版本，使用「重建表」模式：CREATE TABLE new_table → INSERT INTO new_table SELECT ... → DROP TABLE old_table → ALTER TABLE new_table RENAME TO old_table

重写 queries.go（1690 行），系统性替换所有 pgx 调用为 database/sql 标准库（per D-10, D-11, D-12, D-13）：

**1. 导入变更：**
- 移除 `github.com/jackc/pgx/v5` 和 `github.com/jackc/pgx/v5/pgxpool`
- 添加 `database/sql`
- 添加 `github.com/google/uuid`
- 保留 `context`, `encoding/json`, `errors`, `fmt`, `time`

**2. Repository 结构体（per D-10）：**
```go
type Repository struct {
    db *sql.DB
}
func New(db *sql.DB) *Repository {
    return &Repository{db: db}
}
```
方法签名保持不变（per D-10）。

**3. 查询调用替换（per D-11）：**
- `r.db.Query(ctx, sql, args...)` → `r.db.QueryContext(ctx, sql, args...)`
- `r.db.QueryRow(ctx, sql, args...).Scan(...)` → `r.db.QueryRowContext(ctx, sql, args...).Scan(...)`
- `r.db.Exec(ctx, sql, args...)` → `r.db.ExecContext(ctx, sql, args...)`
- `r.db.Ping(ctx)` → `r.db.PingContext(ctx)`
- `r.db.Begin(ctx)` → `r.db.BeginTx(ctx, nil)`

**4. 占位符替换：**
- 所有 `$1, $2, $3...` → `?, ?, ?...`（SQLite 使用顺序 ? 占位符）
- 注意：多处 INSERT 有 5-10 个参数，确保 ? 数量与参数数量一致

**5. 错误类型替换：**
- `pgx.ErrNoRows` → `sql.ErrNoRows`（所有 `errors.Is` 检查）
- 包括 queries.go 底部的 `DeleteClaudeAccountTx` 函数

**6. UUID 处理（per D-13）：**
- 所有 `id::text` 类型转换 → 直接 `id`（SQLite TEXT 列无需转换）
- INSERT 语句中缺少 id 值的：添加 `uuid.NewString()` 作为 id 参数
- 现有 INSERT 已有 id 参数的：保持不变
- 常见模式：`INSERT INTO table (id, ...) VALUES ($1, ...)` — 如果 $1 原来是 DEFAULT gen_random_uuid()，现在需要显式传入 uuid.NewString()

**7. BeginTx 和事务函数：**
- `func (r *Repository) BeginTx(ctx context.Context) (pgx.Tx, error)` → `func (r *Repository) BeginTx(ctx context.Context) (*sql.Tx, error)`
- `func LockClaudeAccountForDelete(ctx context.Context, tx pgx.Tx, id string)` → `func LockClaudeAccountForDelete(ctx context.Context, tx *sql.Tx, id string)`
- `func DeleteClaudeAccountTx(ctx context.Context, tx pgx.Tx, id string)` → `func DeleteClaudeAccountTx(ctx context.Context, tx *sql.Tx, id string)`
- `tx.Exec(ctx, ...)` → `tx.ExecContext(ctx, ...)`
- `tx.QueryRow(ctx, ...).Scan(...)` → `tx.QueryRowContext(ctx, ...).Scan(...)`

**8. SQL 常量更新：**
- 包级 `const ...SQL = \`...\`` 中的占位符全部 $N → ?
- 移除所有 `id::text` → 仅 `id`
- 移除 `NOW()` → `CURRENT_TIMESTAMP`（如果有用到）

**9. Scan 兼容性：**
- database/sql 的 `rows.Scan()` 接受 `*string`, `*time.Time`, `*int`, `*float64`, `*[]byte` 等
- `time.Time` 列存储为 TEXT（ISO 8601），database/sql 默认不自动解析
- 需要注册 SQLite 驱动的时间解析器，或在 Scan 时使用 `*string` 再手动解析
- 推荐方案：在 app.go 连接初始化时注册自定义时间解析，或在 repository 的 Scan 中使用 sql.NullString 接收再转换
- 简化方案：Scan 目标使用 `*string` 接收时间，然后 `time.Parse(time.RFC3339, str)` 转换为 `time.Time`
- 注意：*time.Time 的 Scan 在 modernc.org/sqlite 中可能需要特殊处理，建议用 *string 过渡

**10. 测试文件适配：**
- 所有 `*_test.go` 文件：如果使用 testcontainers PG，改为内存 SQLite
- 测试中 `pgx.ErrNoRows` → `sql.ErrNoRows`
