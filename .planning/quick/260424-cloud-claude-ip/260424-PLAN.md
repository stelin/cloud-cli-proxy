### 结构体设计
```go
type SessionDashboard struct {
    Username        string   // cfg.Username
    ClaudeAccountID string   // mountCfg.ClaudeAccountID（或 authResp）
    ExitIP          string   // 容器出口公网 IP（SSH 查询）
    ImageVersion    string   // 容器镜像版本
    MountMode       string   // 实际挂载模式（full / hot-only / sshfs-only）
    SessionName     string   // tmux session 名（若已知）
    ConflictCount   int      // 热同步冲突数
    OversizedCount  int      // 跳过大文件数
    PromotionCount  int      // 冷文件晋升数
    DowngradeChain  []string // 降级链摘要（如 "auto → hot-only"）
    Warnings        []string // 动态警告列表
}
```

### CollectDashboard 函数签名
```go
func CollectDashboard(
    conn *ssh.Client,
    cfg *Config,
    mountCfg MountConfig,
    snap *LastSessionSnapshot,
) *SessionDashboard
```

实现细节：
1. **出口 IP 查询**：通过 `conn.NewSession()` 执行 `curl -s --max-time 5 https://api.ipify.org 2>/dev/null || echo timeout`，超时 5 秒，失败显示 "timeout" 或 "unavailable"。
2. **账号信息**：`Username = cfg.Username`，`ClaudeAccountID = mountCfg.ClaudeAccountID`（若为空显示 "未绑定"）。
3. **挂载模式**：`MountMode = snap.ActualMode`（若 snap 为空则显示 mountCfg.Mode.String()）。
4. **会话名**：从 `snap.TmuxSession` 取，若空则尝试用 `buildTmuxSessionName(mountCfg.ClaudeAccountID, mountCfg.Cwd)` 计算。
5. **文件状态**：
   - `ConflictCount = snap.ConflictCount`
   - `OversizedCount = len(snap.OversizedFiles)`
   - `PromotionCount = snap.PromotionCount`
6. **降级链**：把 `snap.DowngradeChain` 格式化为简洁字符串列表（如 `["full → hot-only (mergerfs 不支持)"]`）。
7. **Warnings 动态生成**：
   - `snap.ConflictCount > 0` → `"⚠ %d 个文件同步冲突"`
   - `len(snap.OversizedFiles) > 0` → `"⚠ %d 个大文件被跳过（>50MB）"`
   - `len(snap.DowngradeChain) > 0` → `"⚠ 挂载发生降级"`
   - `mountCfg.IsSecondaryClient` → `"⚠ 当前为 secondary 客户端（热同步只读）"`

### Print 方法签名
```go
func (d *SessionDashboard) Print(w io.Writer, noColor bool)
```

渲染格式（紧凑单面板，类似 envcheck.go 但更简单）：
```
╭──────────────────────────────────────────╮
│         Cloud Claude 会话信息面板         │
╰──────────────────────────────────────────╯
  出口 IP          1.2.3.4
  使用账号         alice (claude_abc123)
  镜像版本         v3.2.3
  挂载模式         full
  会话名           claude-abc123-deadbeef
  同步冲突         0
  跳过大文件       0
  冷文件晋升       12

  ⚠ 挂载发生降级（full → hot-only）
```

着色规则（noColor=false 且 w 是 TTY 时）：
- 标题：Cyan
- 出口 IP：Green（成功获取时）
- 警告行：Yellow
- 降级链存在时 MountMode：Yellow

注意事项：
- 所有 SSH 命令必须带 `defer sess.Close()`。
- IP 查询超时 5 秒，失败不 panic，写入 "timeout"。
- 输出到 `w`（即 stderr），使用 `fmt.Fprintln` / `fmt.Fprintf`。
- 不要引入新依赖，复用现有 `Colorize` / `ColorEnabled`。

cleanupMount, finalMode, mErr := MountWorkspace(connA, connB, mountCfg)
if mErr != nil {
    return 0, fmt.Errorf("文件映射失败: %w", mErr)
}
defer cleanupMount()

// 读取 last-session.json 获取 snapshot 数据（MountWorkspace 已写入）

### 注意点
1. `MountWorkspace` 返回值从 `_, _, mErr` 改为 `cleanupMount, finalMode, mErr` —— `finalMode` 当前 dashboard 不需要，但保留以便后续扩展。
2. `LoadLastSession()` 是 `last_session.go` 已有函数，直接复用。

   - **方案**：`SSHConfig.User` 就是用户名，直接用。`ClaudeAccountID` 已在 `mountCfg` 中。
4. 保持 `dash.Print` 输出后紧跟一个空行，与后续 OAuth 检查 / Claude 启动之间留视觉间隔。
5. 如果 `CollectDashboard` 内部 SSH 查询超时（5 秒），不影响后续流程 —— 这是非阻塞的 best-effort 展示。
