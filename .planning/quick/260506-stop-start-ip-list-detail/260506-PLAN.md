## 根因结论（来自 debug 档案）

**根因 A（IP 泄漏）**：user 容器在 restart 路径下保留 bridge 接口，default route 被 docker daemon / Docker Desktop 异步刷新覆盖为 bridge gw，configureWorkerEgress 的改动丢失。sing-box 本身工作正常。

**根因 B（状态机分裂）**：
- list（admin_hosts.go:57-78）注入 DockerStatus，UI 优先以 docker===running 显示「运行中」
- detail 只读 DB host.status，UI 把 failed 渲染成「失败」+「启动」按钮
- worker.Execute 错误路径 UpdateHostStatus("failed") 但不停容器

**修复方案**：
- Fix A：所有平台统一 disconnect bridge（去掉 runtime.GOOS=="linux" 限制）+ configureWorkerEgress 加 verify+retry
- Fix B：list 去掉 DockerStatus 注入，只读 DB status；worker Execute 失败路径补 docker stop
- Fix C：PrepareHost 完成后 smoke verify，IP 不符 teardown + ErrEgressIPMismatch

  ```go
  func verifyWorkerEgress(ctx context.Context, workerName, expectedIP string) error {
      cmd := exec.CommandContext(ctx, "docker", "exec", workerName, "sh", "-c",
          fmt.Sprintf("curl -s --max-time 5 https://api.ipify.org || echo ''"))
      out, err := cmd.Output()
      if err != nil {
          return &NetworkError{Type: ErrEgressIPMismatch, Message: "curl failed: " + err.Error(), HostID: workerName}
      }
      actual := strings.TrimSpace(string(out))
      if actual != expectedIP {
          return &NetworkError{
              Type:    ErrEgressIPMismatch,
              Message: fmt.Sprintf("egress IP mismatch: expected %s, got %s", expectedIP, actual),
              HostID:  workerName,
              Metadata: map[string]any{"expected": expectedIP, "actual": actual},
          }
      }
      return nil
  }
  ```
- 注意：使用 `api.ipify.org`（纯文本返回 IP，无 JSON 解析风险）；`--max-time 5` 防止 hang；curl 失败返回空字符串，与 expectedIP 比对会触发 mismatch
