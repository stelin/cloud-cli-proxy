### Task 1: 全面修订中文主文档（README.md + docs/zh/guide/quickstart.md）

**files:**
- `README.md`
- `docs/zh/guide/quickstart.md`

**action:**
1. 在 cloud-claude CLI 使用章节补充以下新增功能：
   - `cloud-claude doctor [domain] [--fix]` — 五维度自检（network/auth/ssh/mount/disk），带 `--fix` 自动修复
   - `cloud-claude explain <CODE>` — 查询错误码详细说明
   - `cloud-claude sessions` — 列出当前 tmux 会话；`cloud-claude sessions --attach <id>` — 接管已有会话
   - `--new-session` — 强制新建独立 tmux 会话（默认 attach 已有会话）
   - `--take-over` — 接管主会话并踢掉其他客户端
   - `--mount-mode` — 指定映射模式（auto / full / mutagen-only / sshfs-only，默认 auto）
   - `CLOUD_CLAUDE_NO_PROMOTION=1` — 禁用冷文件读触发晋升（Linux 默认启用）
2. 修正"用户自助面板"为"管理后台"，删除"用户可查看主机状态、重建主机、访问 VNC 桌面"等过度承诺（当前无独立用户自助面板，只有 admin 后台）
3. 补充断线重连说明：cloud-claude 内置 Reconnector，网络抖动 30s 内自动恢复，输入缓冲不丢失
4. 补充大文件熔断：HotSync 默认跳过 >50MB 单文件，由 cold 路径兜底，可在 config.yaml 调 `hot_sync_max_file_mb`
5. 修正 docker compose 命令：`docker compose pull`（去掉 `--policy always`，该 flag 仅在较新 docker compose 版本支持，易造成兼容性困惑）
6. 更新功能特性列表：
   - 保留已有正确条目
   - 新增"错误码自解释系统"、"tmux 多端会话管理"、"网络抖动自动恢复"、"大文件映射熔断"、"doctor 五维度自检"
   - 删除"用户自助面板"条目
7. 更新架构组件表，增加 cloud-claude CLI 行
8. 更新 docs/zh/guide/quickstart.md 中与 README 重复的内容，保持同步

**verify:**
- README.md 和 docs/zh/guide/quickstart.md 中均包含 doctor / explain / sessions 子命令说明
- "用户自助面板"字样在 README.md 和 quickstart.md 中不存在
- docker compose 命令不含 `--policy always`
- cloud-claude 使用章节包含 --new-session、--take-over、--mount-mode 说明

**done:**
- README.md 和 docs/zh/guide/quickstart.md 修改完成并通过 diff 自检
